# 质量审查

> 重写合并完成后，按以下维度逐项审查。**如果用 `scripts/extract-manual.ts`，第 3-5 节的检查已自动满足**。

## 1. 内容完整性

- 概述：是否包含功能配置表？
- 保护原理：是否包含各保护动作条件（动作方程 / 逻辑条件 / 方向元件 / 异常告警 / 采样异常）？
- 定值说明：是否包含参数表 + 控制字 + 软压板？

## 2. 排除正确

- 无测控参数？
- 无装置设置？
- 无人机接口 / 通信 / 硬件 / 调试 / 运行监视？
- 第2、4、6、7、8、9 章已整章删除？

## 3. 格式（脚本自动）

- **标题层级正确**：`#` 标题 / `## N` 章 / `### N.M` 节 / `#### N.M.K` 小节
- **无 `<details>` 块残留**（flowchart + text_image 全部清掉）
- **无 `## 1）...` 误标列表项**（应还原为 `1）...`）
- **HTML 表格全部转 markdown**（rowspan/colspan 已展开）
- **无特殊字符列表标记**（Wingdings、行首点等已转 `- `）
- **LaTeX 公式保留**（`$$...$$` 和 `$...$`）

## 4. 图片（脚本自动）

- 引用路径正确（`![](images/HASH.jpg)` 相对路径）？
- **images/ 目录里的图片数 == md 里的图片引用数**（脚本只复制被引用的）
- md 中无残留死链接
- `grep '<details>' md` 返回 0
- `grep 'mermaid' md` 返回 0（除非例外保留）

## 5. 原版 PDF 溯源

- 输出目录里有原版 PDF 文件（`{型号}*.pdf`）？
- 没有的话，校核时遇到 md 和图片对不上的问题无法核实——建议补上

## 6. 已知数据瑕疵（脚本无法修复，需人工）

| 瑕疵 | 原因 | 修复方式 |
| --- | --- | --- |
| 5.1.2 表"1617"序号合并 | 源 HTML 损坏，`<td>16</td><td>17</td>` 在同一 `<tr>` | 手动改源 `full.md` 拆开 |
| 5.4 表"出口名称保护动作"列名合并 | 源 PDF 列名提取错 | 手动改源 `full.md` 拆为两列 |
| colspan=N 注释行重复 N 次 | colspan 展开副作用 | 接受现状，或改源为单格 |
