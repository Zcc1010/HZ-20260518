# COMTRADE 录波文件解析工具集（全电压等级 · 配置自适应）

> 本技能**一个入口覆盖全部电压等级与配置形态**：220kV+（双套，每端 2 套，线路共 4 套）/ 110(66)kV（单套，双端各 1 套或单端 1 套）/ 35kV / 10(6)kV（单套，常两相 CT 配置）/ 配电设备（电容器、电抗器、站用变）。
> **自动开展分析的核心机制 = 电压等级开关**：解析前先识别"电压等级 + 套别 + 接地方式 + 接线方式 + 设备类型 + CT 配置"，据此启用或跳过简报相应章节（详见 `references/电压等级保护配置知识.md` §9 开关速查）。
> 关键红线：单端单套装置若记录两套电流，二者均为本端本地采样，**绝非本侧/对侧**，禁止当作纵差差流（见 §可信度原则 与知识库 §6.1）。

## 配置识别（电压等级开关前置）

**本步骤是"自动按电压等级开展分析"的核心，必须在解析和生成简报前完成。** 从目录结构、HDR、CFG 识别以下维度，写入简报"故障前置条件"：

1. **电压等级**：PT 变比 / 线路额定电压 / HDR 系统参数 → **500（含 750/1000kV 特高压）/ 220 / 110 / 66 / 35 / 10(6) kV**。
2. **套别（配置数）**：
   - **500kV+（含 750/1000kV）与 220kV+ → 双套**（每端 2 套，线路共 4 套：厂站1/2 × 第1/2套）。两者差别在接线与保护配合：**500kV+ 以 3/2 断路器为主、重合闸一般不投（若投逻辑在开关保护）、无常规母差**，须独立分析开关保护（断路器失灵/死区/三相不一致/充电过流/重合闸/失灵联跳）。
   - 110kV 及以下 → **单套**（每设备 1 套；双端线路每端各 1 套 = 双端单套，共 2 套；单端线路 = 单端单套，1 套）。
   - **单端单套"装置内双套电流"红线**：单端单套装置若记录两套电流（保护电流1/2），按知识库 §6.1 三条硬规则判为装置内本地量（通道名无"对侧/远端"、HDR 无纵联功能、两套 CT 变比相同），**绝非本侧/对侧**，禁止当作纵差差流或"穿越性正确"判据。
3. **接地方式**：大电流接地（220/110/66kV，直接接地）/ 小电流接地（35/10kV，消弧线圈/电阻/不接地）。
4. **接线方式**：3/2 断路器（一个半断路器，500kV+ 为主）、4/3、双母双分段、双母、单母分段、线变组、内桥、单母 → 决定是否启用母差章节（线变组/内桥/35/10kV 单母一般无母差；**3/2 接线无常规母差**，改以失灵/死区/联跳替代，并启用开关保护分析）。
5. **CT 配置**：三相 CT / 两相 CT（早期 10kV，仅 A、C 相，B 相无 CT → B 相电流恒为 0，不代表无故障）。
6. **保护功能**：是否含距离保护（110kV+ 有，35/10kV 无）、重合闸（220/110kV 投；**500kV+ 一般不投，若投逻辑在开关保护/断路器保护而非线路保护**）、**开关保护**（断路器失灵/死区/三相不一致/充电过流/重合闸，仅 3/2 及部分 220kV 接线启用，见设备类型"开关保护"）。

> 配置识别结果决定"电压等级开关"：低电压等级**不得**误用高电压等级模板逻辑（如 10kV 出现"距离Ⅰ段不动作判别""重合闸动作"，或单端单套把装置内双电流当本侧/对侧差流）。

## 目录名称解析（厂站 / 设备名称 / 保护套别 推断）

录波归档常按"事故文件夹 / 厂站 / 设备描述 /"组织，**设备描述目录名即编码了厂站外的全部关键信息**（电压等级、设备类型、设备名称、保护套别、装置型号；3/2 接线还能按断路器编号判中/边）。本技能用 `scripts/parse_folder_name.py` 在**配置识别前置步**自动解析，避免逐回人工分辨。

- 用法、字段、命名规律、真实案例、解析规则、3/2 中边判定：见 [目录命名解析](references/目录命名解析.md)。
- 典型命令（解析整个事故文件夹，输出结构化 JSON / 表格）：

```bash
python <skill>/scripts/parse_folder_name.py "事故文件夹/" --json
```

- 输出 `{厂站, 设备名称, 设备类型, 保护套别, 电压等级, 装置型号, 中/边}` 表；凡目录名无法确定的项（如断路器目录常不含电压）一律列入 `needs_review` 标"待核实"，最终以 HDR 校验（保持"HDR内容 > 目录名称 > 用户确认"）。
- 与配置识别关系：解析结果作为"配置识别"的**目录侧输入**，与 HDR/CFG 的 PT 变比、线路额定电压等交叉确认电压等级与套别。
- 3/2 接线中边：完整串按**断路器编号末位**判定——**X2=中断路器、X1/X3=边断路器**；以 HDR 重合闸时序（边先合、中后合）互验。

## 目录结构要求

**双套（220kV+，含 500kV+；3/2 接线时每串含 2 台断路器，各有独立开关保护）结构**：

```
事故文件夹/
├── 保护录波/
│   ├── 厂站一/
│   │   ├── 第1套/
│   │   │   └── *.cfg/dat/hdr/zip（可混合）
│   │   └── 第2套/
│   └── 厂站二/
└── 故障录波/（必须解析，用于一致性评价）
```

**单套（110kV 及以下）结构**：

```
事故文件夹/
├── 保护录波/
│   ├── 本端厂站/（双端单套：两端各一个目录；单端单套：仅本站一个目录）
│   │   └── *.cfg/dat/hdr/zip
│   └── 对端厂站/（双端单套时存在）
└── 故障录波/（可选，双端时两端各一份）
```

> 单套无"第1套/第2套"概念；双端单套每端各 1 套（共 2 个装置），单端单套仅 1 个装置。

## 工作流程

> **⚠ 流程红线**：以下 9 步必须严格按顺序执行。**不得**因案例简单、数据看起来完整、手动读取更快等任何理由跳过任何一步。跳过步骤 0（parse_folder_name）会导致厂站/电压等级识别错误；跳过步骤 5（Subagent 段落）会导致主 Agent 直接读原始数据、丧失数据层与报告层的隔离保护。**绵动线案例为证——三次分析三种结果，根因全在跳步。**

```
┌─────────────────────────────────────────────────────────┐
│ 0. 配置识别（前置）    │ 先用 parse_folder_name.py 从目录名推断 厂站/设备/套别/电压/型号（见"目录名称解析"章），再结合 HDR/CFG 交叉确认
├─────────────────────────────────────────────────────────┤
│ 1. 目录结构检查         │ 确认符合 保护录波/厂站/套别（或单套端别）结构（可用 parse_folder_name.py 自动发现设备文件夹）
├─────────────────────────────────────────────────────────┤
│ 2. 设备类型识别         │ 线路/主变/母差/开关保护（断路器保护）/配电设备 → 选择对应模板
├─────────────────────────────────────────────────────────┤
│ 3. extract_comtrade.py  │ 压缩包 → cfg/dat/hdr文件
├─────────────────────────────────────────────────────────┤
│ 4. parse_dat_to_csv.py  │ CFG+DAT → 原始采样(CSV)
├─────────────────────────────────────────────────────────┤
│ 5. calculate_rms.py     │ CSV → RMS统计 + 一/二次突变 + 事件
├─────────────────────────────────────────────────────────┤
│ 6. calculate_fault_      │ CSV → development.json（分阶段RMS/相关系数/关键事件/判读）
│    development.py        │
├─────────────────────────────────────────────────────────┤
│ 7. 故障录波处理         │ 解压+解析+RMS+事件（必须执行，如有）
├─────────────────────────────────────────────────────────┤
│ 8. Subagent生成段落     │ 每套/每端装置 → Markdown段落（隔离无幻觉）
├─────────────────────────────────────────────────────────┤
│ 9. 主Agent生成简报      │ 只读段落 → 完整简报（按电压等级开关启用/跳过章节）
├─────────────────────────────────────────────────────────┤
│ 10. 隐患规则分析（可选） │ 基于解析结果执行规则判定
└─────────────────────────────────────────────────────────┘
```

**流程约束**：
- 必须先完成段落生成，再生成简报，不得跳过段落步骤
- 主Agent只读段落文件，不得回头读取原始HDR/RMS/EVENTS文件
- 若某套装置段落缺失，在简报中标注`[第X套数据缺失]`（单套标`[本端/对端数据缺失]`）

## 可信度原则

**贯穿始终的核心原则：只写有依据的数据，不确定的数据必须标注。**

### 1. 数据溯源

- 每一项数据必须明确来源（来自哪个段落、哪个文件）
- **不得**在无依据的情况下补充数据
- 段落未覆盖的信息，从目录结构或文件名推断时，须注明

### 2. 跨源交叉验证

| 校验项 | 规则 | 异常标注 |
|--------|------|----------|
| 测距 | 双端测距之和 ≈ 线路全长（±20%） | `[待核实: 测距偏差过大]` |
| 差流 | 双端差流一次值偏差 < 10% | `[待核实: 差流偏差较大]` |
| 动作时间 | 同装置/同厂站动作时间差 < 20ms | `[待核实: 动作时间差较大]` |
| 电流一致性 | 保护录波与故障录波同相电流偏差 < 20% | `[待核实: 电流一致性存疑]` |

> **测距验证说明**：双端测距中，本端测距+对端测距≈线路全长为正确。单端单套时本端测距 ≤ 线路全长（定值/台账）且合理即可，不强行"本侧+对侧≈全长"。
>
> **两侧电流差异说明**：线路两侧故障电流可能差异较大，原因是两侧电源系统阻抗不同、故障点距两侧电气距离不同导致的分流不同，属于正常现象。

### 3. 差流一次值校验

生成线路保护简报时，对差流一次值进行交叉校验（**仅双端配置：双套 220kV+ 或 双端单套 110kV±**；**单端单套无差流分析**）：

- **双套（220kV+）**：同侧两套差流一次值应一致（<10%）；两侧四套差流一次值应基本相等（<10%）。
- **双端单套（110kV±，2 装置）**：两端差流一次值应基本相等（<10%）。
- 若不一致必须标注 `[待核实: 差流偏差较大]`。

计算公式：一次值(A) = 差流二次值(A) × CT变比(一次额定/二次额定)

> **Why**: 同一故障点，同一时刻，双端保护看到的差流应是同一个电流值（穿越性为0），若偏差过大说明计算有误或装置采样异常。
>
> **单端单套"装置内双套电流"红线（必读）**：单端单套装置若记录两套电流（保护电流1/2），二者均为本端本地采样，**绝非本侧/对侧**——通道名无"对侧/远端"、HDR 无纵联功能项、两套 CT 变比相同，三者任一即判为同端本地量。其差值仅反映双套采样/变比一致性，**禁止**当作"差流≈0、呈穿越性、纵差未动正确"的判据。详见知识库 §6.1。

### 4. 动作元件一致性校验（双重化配置专用）

> **双重化 = 同一厂站同一被保护设备配置两套独立保护**（典型 220kV+ 线路双套、500kV+ 3/2 线路双套；母线/主变按厂家双重化配置）。
> **非双重化（110kV± 单套、10/35kV 单套、单端单套）→ 本节 N/A，不输出两套一致性表格**；单套分析侧重"动作元件与定值是否匹配、保护是否正确动作"。

在"两套保护动作一致性"分析中，必须校验以下所有动作元件：
1. **纵联差动保护**：动作时间、返回时间、持续时间
2. **接地距离Ⅰ段**：动作时间、返回时间（若多套装置配置，需对比）
3. **接地距离Ⅱ段及以上**：动作时间、返回时间
4. **工频变化量距离保护**：动作时间
5. **零序过流保护**：动作时间
6. **距离加速/零序加速**：动作时间
7. **重合闸动作/返回**：动作时间

**一致性判定规则**：
- 动作时间差 > 20ms → `[待核实：动作时间不一致]`
- 返回时间差 > 20ms → `[待核实：返回时间不一致]`
- 持续时间差 > 10ms → `[待核实：持续时间差XXms>10ms]`

**一致性结论判定**：
- 任一动作元件出现上述不一致 → 该厂站两套保护判为"不一致"
- 所有动作元件均满足误差范围 → 该厂站两套保护判为"一致"

> **注意**：一致性表格中的"一致性"列应直接给出结论，一致时填"一致"，不一致时填具体待核实原因。
> **单套（110kV 及以下）无此节**：单套无"两套"概念，删除同侧两套一致性分析，改用"双端单套两端差流一次值比对"或"单端单套测距 vs 全长"。

### 5. 不确定数据的处理

- **文件缺失** → 标注 `[本套文件缺失]` 或 `[录波文件缺失]`
- **字段无数据** → 标注"无数据"，不得留空
- **数据矛盾** → 标注 `[相别矛盾]` / `[测距异常]` 等，不强行统一
- **数值超出合理范围** → 标注具体疑问，不得自行修正

### 6. 段落模板的核对步骤

Subagent 生成段落后，必须按模板中的"核对步骤"自检，不一致时明确标注。

### 7. 距离Ⅰ段不动作原因分析（仅 220/110(66)kV 线路）

线路保护中，接地距离Ⅰ段未动作是常见现象（故障点多在Ⅰ段保护范围外），但在跳闸简报中**必须**判别"未动作是否正确"，避免将"应动未动"误判为"正常不动作"。**35/10kV 无距离保护，本节 N/A。**

**触发条件**(满足任一即触发分析):
- 距离Ⅰ段在相关装置中均无动作信号，但故障测距 < 距离Ⅰ段有效保护范围(疑似应动未动)
- 用户/调度质疑距离Ⅰ段为何未动作
- 故障判定结论章节将"距离Ⅰ段未动"列为待核实项

**必查三项**(Subagent 段落生成时输出,主 Agent 集成到简报):

| 必查项 | 数据来源 | 判别规则 |
|--------|----------|----------|
| ① 控制字 | HDR `<DigitalStatus>` 查"距离保护Ⅰ段有效" | =0 → Ⅰ段被退出(用户选择),不动作**正确**<br>=1 → Ⅰ段已投入,需判别保护范围 |
| ② 有效保护范围 | 接地距离Ⅰ段定值(Ω) + 零序电流补偿 + 线路参数 | 范围(km) ≈ Ⅰ段定值 / Z1(Ω/km) |
| ③ 故障点距离 | HDR FaultInfo 测距 / 双端测距之和校验 | 故障点 > Ⅰ段范围 → 不动作**正确**<br>故障点 ≤ Ⅰ段范围 → Ⅰ段**应动未动**,须按隐患规则处理 |

**结论模板**(在主 Agent 简报 §"六、综合分析" 之后以独立子章节或注释块呈现):

```
### X.X 距离Ⅰ段不动作原因判别

| 判别项 | 值 | 结论 |
|--------|----|------|
| 控制字(距离保护Ⅰ段有效) | 0/1 | {退出/已投入} |
| Ⅰ段定值 | X.XX Ω | - |
| Ⅰ段有效保护范围 | X.XX km | (按 Z1=X.XX Ω/km 推算) |
| 本端/各端测距 | X.XX km | - |
| 故障判定 | 故障点 vs Ⅰ段范围 | {在范围内(应动未动) / 在范围外(正确不动作)} |

**结论**: 距离Ⅰ段不动作 {正确 / 待核实: 应动未动}。
```

**案例**: 2026-06-28 明惠2V38线 A 相永久接地故障:
- 控制字 "距离保护Ⅰ段有效"=0 → 退出(用户配置)
- Ⅰ段定值 0.25Ω,Z1=0.072+j0.276Ω/km → 范围 ≈ 0.7 km
- 本端测距 2.40-2.33 km,远大于 0.7 km
- **结论**: 不动作正确(即使投入也不应动作,故Ⅰ段未投亦不影响系统安全)

> **Why**: 简报中将"应动未动"误识为"正确不动作"是常见错误,本节通过 3 项必查+结论模板强制 Subagent 输出关键参数,主 Agent 据此判别,避免跳过此流程。

## 通用分析规则

### 装置参数必提取清单（每台装置解析后第一步，在结论前完成）

**每台保护装置解析后，必须从 HDR 提取以下参数**（不得跳过，不得出具结论后再补）：

| 参数 | 来源 | 用途 |
|------|------|------|
| CT一次额定值 / CT二次额定值 | HDR SettingValue | 故障电流一次值换算、差流一次值互校 |
| PT一次额定值 | HDR SettingValue | 故障电压一次值换算 |
| 线路总长度 | HDR SettingValue | 测距验证（双端测距和≈全长） |
| 差动动作电流定值 | HDR SettingValue | 差动正确性验证 |
| 接地距离Ⅰ段定值 + 正序阻抗 | HDR SettingValue | Ⅰ段保护范围计算 |
| 线路正序/零序阻抗 | HDR SettingValue | 距离保护范围校验 |

**NSR 装置特殊处理**：NSR HDR 无 `<DeviceInfo>`/无版本号字段，CT/PT 优先从 CFG 第1行（型号+版本）和 CFG 模拟通道比例取出；HDR SettingValue 中也可能有（如 NSR-303A 系列），两处均查。

**500kV 3/2 接线 CT/PT 破防规则**：若双套线路保护 HDR 均无法提取 CT/PT，可参考同一串内任一断路器保护的 CT/PT（3/2 接线中同一线路所连两开关 CT 变比相同）。引用时标注来源（如"取自 5051 PCS-921A HDR"）。

**HDR 无事件的破防规则**：若 HDR 无 TripInfo/DigitalEvent，必须从 CFG 数字通道（开关量）通过 `parse_dat_to_csv.py` + `calculate_rms.py` 提取通道变位事件（events.csv）。香墨线 5052 WDLK862A 即通过此方法从 DAT 开关量成功提取了 B 相跳闸/跳位。

**WXH-803A HDR 差流≠最大值**：许继 WXH-803A HDR FaultInfo 中差流为**保护动作瞬间值**（trip moment snapshot），非故障过程最大值。差流比对时必须统一从 RMS 波形提取差流最大值（通道名含"差动电流"或"差流"的通道 RMSmax），否则会严重低估差流（蓼叶线实测：WXH HDR 4.06A vs RMS max 39.40A，差 ~10 倍）。PCS-931A HDR "最大差动电流"直接为最大值，可采信。南庙线许继 PAC-805GD 母差和许继 WXH-803A 线路保护均适用此规则。

**定值"退出"≠ 保护退出**：HDR SettingValue 中 `XX闭锁YY = 0-退出` 表示 XX 发生时**不闭锁** YY，而非 YY 保护被退出。仅当定值名为 `YY保护软压板 = 0-退出` 或 `YY保护控制字 = 0-退出` 时，"退出"才代表保护本身不可用。判定方法见 `references/电压等级保护配置知识.md` §10。绵动线 PSL-621UDA-G `CT断线闭锁差动 = 0-退出` 被误读即为此规则的反例。

**四方（CSC）开关保护 HDR 软压板位置**：四方 CSC 系列开关保护（CSC-121A、CSC-21A 等）的 HDR 中，重合闸相关软压板（停用重合闸、重合闸充电完成等）**不在 `<SettingValue>` 中作为独立软压板字段**，而是在 `<DigitalStatus>` 中以**实时状态位**形式呈现。解读重合闸功能投退时须同时检查三处：
- `<SettingValue>`：控制字（单相重合闸、三相重合闸、停用重合闸）
- `<DigitalStatus>`：停用重合闸（=1 表示软压板已停用）、重合闸充电完成（=0 表示未充电/不具备重合条件）
- `<TripInfo>/<DigitalEvent>`：实际动作行为验证（低气压闭锁重合闸、重合闸动作等）

> **案例**：涧鹭 5774 线 5052 第1套 CSC-121A，控制字单相=1/三相=0/停用=0，乍看重合功能投入；但 DigitalStatus 停用重合闸=1、重合闸充电完成=0，实际软压板已停用且未充电，重合功能不可用。仅读 SettingValue 会误判为"无软压板字段、投入但未重合"，漏掉真实投退依据。**适用于所有四方 CSC 系列开关保护装置**。

**时序分析 > HDR TripInfo + DAT 开关量合并**：保护动作时序分析必须综合两者——HDR TripInfo 提供保护装置自身记录的动作时刻（精确到 ms），DAT 开关量（events.csv）提供返回时刻、跳位/合位变化、及 HDR 未记录的附加信号。蓼叶线 WXH-803A HDR 仅 5 条 TripInfo（无返回）→ DAT 补齐 17 个事件（含 TWJb 跳位/合位/全部返回时刻），合并后时序完整可互验。

**110kV 及以下 HWJ/TWJ 判定规则**：HWJ（合位继电器）返回不代表开关已跳开——其回路接于合闸辅助触点，返回可能仅表示合闸回路中断。**TWJ（跳位继电器）由 0→1 才是开关跳开的确认信号**（接于跳闸位置辅助触点，与主触头机械联动）。分析时序时须以 TWJ 变位为开关状态确认依据，HWJ 变位仅作辅助参考。同理，开关合闸确认以 HWJ 由 0→1 或 TWJ 由 1→0 为准（而非仅看合闸出口脉冲）。详见 `references/电压等级保护配置知识.md` §6.1。

> **Why**：本清单系 香墨线 500kV 案例教训总结——初始分析时仅读了 TripInfo/FaultInfo（动作事件/故障量），未读 SettingValue（装置参数），导致 CT/PT 列为空、一次值未换算、距离Ⅰ段范围无法计算。后续每台装置必须"先读参数、再读事件"。

### 装置型号来源

- **优先取自保护录波 HDR `<DeviceInfo>` 中的装置型号**——HDR 是保护装置自身生成的故障报告，Device Info 中的型号为装置完整型号（如 `CSC-121A-DG-GCN`），是型号的第一手权威来源。
- 录波文件名（.zwav 原始文件名，GBK 解码后）作为辅助校验，与 HDR 不一致时以 HDR 为准。
- CFG 第1行（COMTRADE 标准头）亦含型号信息，三方互校。**HDR DeviceInfo > 录波文件名 > CFG 第1行**。
- **案例**：涧鹭5774线 5013 开关保护，.zwav 文件名中型号为 `CSC-21A`/`PRS-72NA`（截断），HDR DeviceInfo 为完整体 `CSC-121A-DG-GCN`/`PRS-721NA-DG-GCN`，以 HDR 为准。

### 制造厂家识别

根据装置型号前缀判断厂家：
- **PCS开头** → 南瑞继保
- **NSR开头** → 国电南瑞
- **CSC开头** → 北京四方
- HDR无厂家数据 → 标注`[待核实]`，不自行填充

> **数据来源**：必须从HDR DeviceInfo中获取，不得基于型号推断。若HDR中无厂家数据，标注`[待核实]`。

### 常规站/智能站判断

根据装置型号后缀判断：
- 含 FA/DA/DG → 智能站
- 其他 → 常规站
- 在保护配置表（第二节/第三节）备注列标注"常规站"或"智能站"

### 跳闸相别判断

综合以下信息判断故障相别：
1. 保护动作信号中的相别标注（如"C相"）
2. 电流正突变值最大的那相
3. 电压负突变最明显的相
- 三者一致则确认，不一致则标注矛盾。
- **两相 CT 配置（早期 10kV）**：B 相无 CT → B 相电流恒为 0/派生值，**不代表 B 相无故障或健康**；判相别须**以电压特征为主**（A-B 两相短路→AB 线电压塌缩、BC/CA 近正常；A-C 两相短路→AC 线电压塌缩）。

### 故障电流取值

三源综合定相后，取故障相的RMS最大值（一次值）。

### CT变比获取优先级

1. **HDR SettingValue** - 直接读取CT一次/二次额定值
   - **主变保护**：各侧CT变比须优先从本保护 HDR 提取；若本装置 HDR 缺失（如精简头），可复用同主变另一套定值并标注推定来源（如"依第2套定值推定"）。
2. **对侧电流推断** - 本侧录波中对侧电流RMS最大值 vs 对侧录波中本侧电流RMS最大值，比值推断CT变比倍数关系（双端配置）
3. **差流验证** - 两侧差流应基本相等，验证推断是否合理（双端配置）
4. **无法推断** - 只报二次值，注明"无CT变比信息"

### 同侧两套定值一致性原则（双套 220kV+ 专用）

生成跳闸简报时，同一厂站两套线路保护的定值应满足以下原则：

**一致性要求**：
- CT变比、差动动作电流定值应完全一致
- 若不一致须在报告中标注并说明原因

**定值补充规则**：
- 若某套装置段落中缺少定值信息，可从同侧另一套补充
- 补充的定值须标注 `(*)` 并在简报末尾备注说明来源
- 优先使用本套录波数据中的定值，本套无记录时才从同侧补充

**标注格式**：
- 定值表中：补充的定值标注 `(*)`，如 `0.79(*)`
- 简报末尾须在备注或说明中注明：`(*) 表示该定值根据同侧另一套保护装置定值补充`

**差流计算规则**（双端配置：双套 220kV+ / 双端单套 110kV±）：

差流数据获取优先级（依次尝试）：
1. **HDR中差流/差动电流** — 识别TripInfo或FaultInfo中**标题明确包含**"差动电流""ICD""差流""最大差动电流"等关键词的测量值
2. **录波文件中差动电流/差流通道** — 识别CFG/RMS中**通道名称包含**"差动电流""ICD""差流"等关键词的通道
3. **时标对齐后计算**（备选方案） — 在时标对齐的前提下，故障相两侧电流先分别换算为一次值，再矢量求和计算差流

数据区分：
- "分相差动动作"是**动作元件信号**，其关联的FaultInfo中是故障电流 → 不能使用
- 必须取FaultInfo中`<name>`标题**明确包含**"差动电流""ICD""差流"等关键词的测量值

数据来源标注规则：
- HDR获取：标注如"9.23A（HDR）"或"HDR最大差动电流"
- RMS获取：标注如"13.474A（RMS）"或"RMS通道一A相差动电流"
- 时标对齐计算：标注如"计算值"并说明方法

校验规则：
- 双套（220kV+）：同侧两套差流一次值应基本相等（<10%）；两侧四套差流一次值应基本相等（<10%）
- 双端单套（110kV±）：两端差流一次值应基本相等（<10%）
- 偏差≥10%须标注`[待核实: 差流偏差较大]`

**差流一次值交叉校验表**（双端配置）：

| 厂站/端别 | 套别 | 差流二次值(A) | CT变比 | 差流一次值(kA) |
|------|------|---------------|--------|----------------|
| 厂站1/本端 | 第1套 | X.XXX | CT1/1 | X.XXX kA |
| 厂站1/本端 | 第2套 | X.XX | CT1/1 | X.XXX kA |
| 厂站2/对端 | 第1套 | X.XX | CT2/5 | X.XXX kA |
| 厂站2/对端 | 第2套 | X.XX | CT2/5 | X.XXX kA |

校验结论：偏差<10%为一致，≥10%须标注`[待核实: 差流偏差较大]`

### 差动电流

- 计算：各侧（线路）或各间隔（母差）电流矢量和
- 验证：与录波差流通道值交叉验证
- 穿越性判别：

| 条件 | 差流幅值 | 两侧电流相位 | 判别结果 |
|------|---------|-------------|---------|
| 区内故障 | ≈两侧电流之和 | 同相 | 非穿越性 |
| 区外故障（极性正确） | ≈0 | 反相 | 区外故障，穿越性 |
| 区外故障（极性接反） | ≈两侧电流之和 | 同相 | 非穿越性，可能极性接反 |

> **CT极性接反识别**：若区外故障时差动保护误动，且差流增大但两侧电流相位同相，应检查CT极性是否接反。

### 一次值换算

```
变比 = CT一次额定值 / CT二次额定值
一次值(A) = 二次值(A) × 变比
```

例如：CT变比2500/5，变比=500，二次值29.26A，一次值=29.26×500=14.630kA

CT参数从HDR SettingValue读取。各侧/各间隔CT可能不同（主变各侧、母差各间隔分别读）。

## 突变检测（支持二次）

### 第一次突变

```
正突变1（故障发生）→ 负突变1（故障切除）
```

阈值 = RMS最大值 × 20%（代码 `threshold = 0.2 * max_val`），查找第一个超过阈值的突变。恢复判定阈值 = 阈值 × 0.5（连续 10 个 RMS 差值点绝对值 < 阈值×0.5 视为电流恢复平稳，再检测第二次正突变）。

### 第二次突变（重合闸后）

```
正突变1 → 负突变1 → 电流恢复平稳 → 正突变2（重合后再次故障）→ 负突变2
```

检测起点：电流恢复后再次出现的突变。calculate_rms.py 输出第二次突变值和时间。

## 时间对齐规则（两步对齐法）

### 核心原则

**故障电流正突变是同一物理事件在各装置录波上的表现**。无论装置时钟偏差、CFG 触发机制差异如何，各套保护的故障相正突变绝对时刻指向**同一个故障瞬间**。对齐的目标是将所有装置的事件统一到同一时间轴上，使跨装置时序可直接对比。

### 第一步：电流正突变对齐

1. 找出每个装置故障相电流正突变值最大的一相及其绝对时刻
2. 选择所有装置中正突变时刻最早的为参考时间（即故障绝对时刻）
3. 各装置校正量 = 参考时间 − 该装置正突变时间
4. **无电流正突变的装置**，分两种情况处理：
   - **弱电源侧**（新能源站等）：用**同厂站/同端**第一次负突变进行对齐
   - **录波无预故障数据**（CFG 触发=记录起始，DAT 第 0 采样点起故障电流已越阈值，无法检测正突变）：用**其他装置正突变时刻的平均值**作为该装置的外部故障参考时刻，校正量 = 参考时间 − 该装置保护启动时刻 + 典型启动延迟（≈5ms）

> **验证**：校正后各装置的保护启动时刻应落在同一物理时间窗口内（差异通常 ≤10ms，对应不同装置对同一故障的启动灵敏度差异）。若某装置校正后保护启动偏离其他装置 >50ms，应检查其正突变计算是否有效（蓼叶线 PCS-931A 实测：DAT 无预故障导致正突变不可计算）。

### 第二步：保护启动对齐

1. 将校正量应用到各装置的保护启动时间，得到校正后保护启动时间
2. 选择校正后最早的保护启动时间为 t=0ms 基准
3. 所有事件的相对时间 = 校正后事件时间 − t=0ms 基准时间

> **对齐结果应用**：校正量一旦确定，应用于该装置的**所有事件时间**（包括动作、返回、突变、开关位置变化等）。**严禁**以各装置自身保护启动为独立 t=0 再比较——那会导致同一物理时刻被不同数字标注，遮蔽真实的时序一致性。
> **单端单套（1 装置）**：以本装置电流正突变为 t=0，无需跨装置对齐。

**第零条（流程红线 · 绝对优先）**：

1. **PDF 非默认**：输出默认仅含 `.md` 简报；**除非用户明确要求，否则不得自动生成 PDF**。

6. **保护装置与录波器分表**：§二"涉及保护装置"中，保护装置与故障录波器为两张独立表；保护表列为「序号 / 厂站名称 / 装置名称 / 型号 / 厂家 / 录波启动 / CT/PT」，录波器表列为「序号 / 厂站名称 / 装置名称 / 型号 / 录波启动」。禁止把录波器混入保护装置表。
7. **电气量互校表默认不列录波器**：表中仅列保护装置数据。故障录波器数据仅在**同厂站两套保护电气量不一致**（偏差≥10%）时作为仲裁参考引入，并注明引入原因。
8. **CT/PT 必须提前提取，不得补填**：每台保护装置在出结论前须从 HDR SettingValue（或 NSR CFG）提取 CT一次/二次、PT一次、线路全长、差动定值、距离各段定值，填入装置表。**禁止**在 CT/PT 列为空时出具一次值结论。详见 §通用分析规则 → 装置参数必提取清单。
9. **两侧差流一次值必须互校（双端线路必做）**：对于差动保护动作的线路故障，取各装置 B 相差流**最大值**（WXH-803A HDR 为动作瞬间值不可用，必须从 DAT RMS 提取）乘以 CT 变比换算一次值。两侧一次值应基本相等（偏差 < 10%）。注：需在表中区分"动作瞬间值"和"波形最大值"，防止误将 WXH HDR 差流当最大值比对。偏差 ≥ 10% 须标 `[待核实: 差流偏差较大]`。
10. **§一 故障线路行仅含线路参数**：故障线路行格式为「线路名（厂站1 ↔ 厂站2），电压等级，全长 XX km；线路正序阻抗 XXΩ、零序阻抗 XXΩ」。**不含** CT/PT 变比和接地距离Ⅰ段定值（CT/PT 在 §二 装置表、Ⅰ段在动作行为分析章）。
11. **§一 故障位置含两侧距离及占比**：格式为「距厂站1侧约 XX km（占全长约 XX%），距厂站2侧约 XX km（占全长约 XX%）」。
12. **HDR 无事件时从 DAT 开关量提取**：若装置 HDR 无 TripInfo/DigitalEvent，必须通过 CFG 数字通道 + `parse_dat_to_csv.py` + `calculate_rms.py` 从 DAT 提取 events.csv，不可放弃该装置。
13. **所有保护装置必须 HDR + DAT 双源综合**：每台保护装置的时序表必须综合两个来源——HDR TripInfo（保护逻辑信号：动作时刻、故障量、选相）和 DAT events.csv（波形开关量：返回时刻、HWJb 合位、TWJb 跳位、GOOSE 信号、低气压等辅助信号）。**严禁仅凭 HDR TripInfo 出时序表**，即使 HDR 看似完整（如 PCS-921A 有 8 条 TripInfo），DAT 仍有 HWJb/TWJb 等 HDR 不记录的关键信息。香墨线全部 9 台装置已验证：HDR 仅动作/返回时刻，DAT 补齐了开关机械位置和 GOOSE 网络信号。
    - **时序表每条动作须标注来源**：`[HDR]` 仅 HDR 有、`[DAT]` 仅 DAT 有、`[双源一致]` 两者一致。HDR 独有信号（如差动相电压、失灵相电压、故障信息等保护内部量测逻辑）必须在时序表中体现；DAT 独有的 TWJ/HWJ 机械位置也须显式列出。HDR 与 DAT 同一事件时刻差 >3ms 时分行列并标 `⚡[HDR +Xms / DAT +Yms]`。
14. **正突变统一对齐（同一时间轴）**：所有装置通过**故障相正突变时刻**对齐到同一时间轴——正突变是同一物理故障在各装置上的表现，对齐后统一以最早校正保护启动为 t=0ms。**严禁**各装置以自身保护启动为独立 t=0 后直接排表比较。HDR 与 DAT 同一事件时刻差 ≤3ms 合并为一个值，>3ms 分别标注来源。
15. **差流比对必须统一取 RMS 最大值并换算一次值**：WXH-803A HDR FaultInfo 中差流为保护动作瞬间值（非最大值）。差流必须从 DAT RMS 提取 RMSmax，并换算为一次值（二次值 × CT变比）作为主要展示值，二次值括号附注。PCS-931A HDR "最大差动电流"即为最大值可直接采用。两侧一次值偏差 < 10% 为一致。详见 §通用分析规则。
16. **表格格式简洁优先**：时序表列头用「时刻」不用「相对时刻」；装置列头用型号+厂家简称，不重复完整型号后缀；装置表「序号」→「#」、保持内容紧凑；数据列让 renderer 自动分配宽度，不刻意对等列宽。

17. **双重化主变保护必须开展两套电气量与差流比对**：双重化主变简报须含「两套电气量及差流比对」专节（**单套主变 N/A**），分别比对同厂站两套保护的**各侧电气量**（高压/中压/低压侧三相电流、各侧电压）与**差流**（纵差差流 RMS 最大值，换算一次值或标幺 Ie）：
    - **各侧电流比对**：取故障期间各侧三相电流 RMS 最大值（二次值为主、一次值括号附注），两套同侧同名相偏差 **<20%** 为一致。
    - **各侧电压比对**：取各侧相/线电压（二次值），偏差 **<10%** 为一致。
    - **差流比对**：纵差差流 RMS 最大值，两套偏差 **<10%** 为一致。
    - 任一超阈须标 `[待核实：两套电气量偏差较大]` / `[待核实：两套差流偏差较大]`，并优先依故障录波器（如有）仲裁。
    - 若某套差流通道未命名/无数据（如 HDR 精简、模拟量通道泛化 ACCx），标注 `[第X套差流通道缺失]`，以动作行为（起动/出口）+ 矢量和分析互校，**不得臆造差流值**。
    - **比对目的**：确认两套采样回路一致、故障识别一致。与「动作元件一致性（时间）」互补——本规则侧重**电气量幅值**一致性，是双重化主变简报的强制章节。

18. **「故障发展过程分析」仅复杂故障或发展性故障须开展；单一阶段故障仅需在 §1 简述即可**：主变/线路/母差/配电设备中，**仅当判明为复杂故障或发展性故障时**（如经电弧单相接地→相间→三相、多重故障、越级），方须基于**原始波形 CSV 时序**在「动作时序表」之后设「故障发展过程分析」专节，还原故障**起始→发展→全盛→切除**演化：
    - 提取故障元件三相电压/电流的波形基础（采样率、有无预故障段）、关键事件时刻（故障电流启动/各相电压塌缩/发展转折/切除）、各阶段三相 V/I 的 RMS、全盛期相间相关系数。
    - 判读**是否发展性故障**、**电弧性/稳定性**、保护动作是否"跟住"故障演化。
    - **单一阶段故障**（如稳定单相接地、三相短路、无扩展的相间短路）：**不设专节**，仅在 §1「故障基本情况」表格的"故障发展过程"行填 **"非发展性，单一阶段"** 即可，无需展开分析。小电流接地系统（35/10kV）单相接地初始仅容性电流、电压特征明显，**不得误判为相间短路**；若确为稳定单相接地，亦按单一阶段处理。

> **Why**：嘉山变#2主变案例仅凭"低压1分支复压过流动作"会误判为普通分支故障；而波形还原显示其为"单相接地→B-C相间→三相短路"的发展性故障——该维度直接支撑了"正确动作"结论。屏显变 220kV Ⅱ母 A 相接地则为稳定单一阶段故障，§八·五整节不设、仅 §1 标注"非发展性，单一阶段"。

19. **套别（第1套/第2套 = A套/B套）必须由 HDR/IED 名称判定，严禁按文件遍历顺序赋值**：双重化保护的两套装置，其"第1套/第2套"标识按以下优先级确定，**不得**依据解压/遍历 `.zwav`/`.dat` 文件的顺序（os.walk 顺序随文件系统变化、无任何物理意义）直接给 set1/set2 再映射为第1套/第2套：
    - **第零优先 —— .zwav 原始文件名（GBK 解码后）**：zip 内 .zwav 文件常以 GBK 编码命名，形如 `安徽.屏显变_220kV母线第一套保护PCS915D_...zwav`，文件名直接写明"第一套"/"第二套"+"保护型号"+"站名"。解码回路：`raw_bytes = garbled.encode('cp437'); real_name = raw_bytes.decode('gbk')`（`extract_comtrade.py` 的 `extract_zip()` 已内置此逻辑，解压时自动打印 `[GBK解码]` 日志）。文件名中"第一套/第二套"是**最直接的套别证据**——屏显变案例最初未用此信息致整套颠倒。
    - **第一优先 —— HDR 显式"第X套"标识**：如装置 HDR 出现"`第一套合并单元`"、通道名含"第1套/第2套"、或 DeviceInfo 直接标注套别，以 HDR 为准。
    - **第二优先 —— IED 名称后缀 A/B（电网惯例 A=第1套、B=第2套）**：IED 名形如 `PM220A`→第1套、`PM220B`→第2套；`PL2211`/`PL2204` 等亦遵此（A 序号小者多为第1套）。**IED 后缀与套别的对应是客观判据，优先于任何目录遍历顺序**。
    - **回退**：`parse_folder_name.py` 若从目录名解析不到套别（`set_no=null`），须在 `needs_review` 标注"`套别以 HDR/IED 为准`"，并由主 Agent 在配置识别阶段据 HDR/IED 回填，绝不可把遍历顺序当作套别。
    - **互校**：文件名、IED 后缀、HDR 标识三者应一致；发现矛盾（如文件名写"第二套"但 IED=B→亦第二套、一致无矛盾）时追踪原因。目录顺序与此三者冲突时**以文件名/IED/HDR 为准**整体翻转标签。
    - **易错点**：屏显变 220kV Ⅱ母故障曾因 `.zwav` 遍历顺序 + GBK 解码未做（cp1252→gbk 回路错误，应为 cp437→gbk），把 `PM220B`（国电南自 SGB-750D）误标为第1套、`PM220A`（南瑞继保 PCS-915D）误标为第2套，全套标签颠倒。嗣后一律以"文件名 GBK 解码 → HDR 标识 → IED 后缀"三级互校定套别。

20. **母差简报保护配置须含三子节，母联/分段须精确到具体母段**：
    - **§3.1 保护装置信息表**：套别/型号/厂家/软件版本/IED。
    - **§3.2 母差保护各支路CT变比及运行母段**：列出所有投运间隔的 CT 变比（须从本保护 HDR SettingValue `支路NCT一次值/二次值` 提取，不得仅用基准 CT）、运行母段、故障前运行状态。
      - **母联/分段**须明确到 ⅠA/ⅠB 具体母段（如"ⅠA母 ↔ Ⅱ母"、"ⅠB母 ↔ Ⅱ母"、"ⅠA母 ↔ ⅠB母"），**不得笼统写"Ⅰ母 ↔ Ⅱ母"**——双母/双母单分段下Ⅰ母可能裂为多段。
      - **其余支路**须使用 PCS 三母段刀闸模型（1G=ⅠA、2G=Ⅱ、3G=ⅠB）区分 ⅠA/ⅠB；SGB 仅 1G/2G 无法区分 ⅠA/ⅠB 时，以 PCS 为准标注并注"[建模差异]"。
      - **故障段间隔须加粗⚡标识**。
    - **§3.3 接线与母线运行方式**：判定故障前运行方式——依据 HDR 母联/分段 TWJ/HWJ + 互联/分列软压板信号，判定为**正常并列运行** / **互联运行** / **分列运行**。正常并列简要说明即可；互联/分列须详述影响及差动判据变化。
    - 模板位置：`references/母差跳闸简报模板.md` §3.1~§3.3。屏显变为首个落地案例。

21. **所有保护录波分析须在「故障发展和保护动作情况」节末尾提示故障期间的告警/异常信号**：须检查各装置 HDR（DeviceInfo/SettingValue/DigitalStatus）和 DAT events.csv 中是否存在故障期间发出的**告警或异常信号**（CT断线、PT断线、装置故障、通信异常、采样无效、SV/GOOSE 断链、长期启动、不一致等）。无告警则写"**故障期间无告警或异常信号**"并附 CT断线告警定值等监控阈值供参考；有告警则逐条列出信号名称、时刻，并标注对保护动作的影响（是否闭锁、是否影响正确动作评价），列入待核实问题。

22. **§一 故障基本情况表不含接线方式和接地方式**：接线方式（3/2、双母等）在 §三 保护配置说明中体现；接地方式在 §6.6 零序逻辑中体现，均不在 §一 表中列出。

23. **接地距离Ⅱ/Ⅲ段时间两侧不同属正常配置，不解释**：线路两侧接地距离Ⅱ/Ⅲ段时间定值常因保护配合级差而不同，属正常现象。只需在定值表中列出两侧时间值，**不得附加"这是正常配合"等多余解释**。

24. **非发展性故障严格不设故障发展过程分析专节**：执行规则 #18——单一阶段故障仅在 §1「故障基本情况」表填"非发展性，单一阶段"，不得设立任何形式的专节；非发展性故障的简要描述已在 §四「故障发展和保护动作情况」覆盖，不需额外章节。

25. **多母段接线母差须按母段分别输出小差（双母单分段必列 ⅠA母/ⅠB母小差）**：母差简报 §七 差流分析对**多母段接线**——双母单分段（ⅠA/ⅠB/Ⅱ）、双母双分段（ⅠA/ⅠB/ⅡA/ⅡB）——**必须按母段分别列出小差**，ⅠA母小差、ⅠB母小差、Ⅱ母小差（双母双分段加 ⅡA/ⅡB 母小差）**各自独立成行**，**严禁**合并为单一"小差（故障段）"或笼统的"Ⅰ母小差"。理由：① 非故障段（如双母单分段下的 ⅠA/ⅠB 母）小差应≈0，是验证母差**选择性**、证明未误跳其他母段的关键判据；② 合并输出会掩盖非故障段小差异常（某段小差偏大提示该段 CT 极性/刀闸位置/建模错误）；③ 故障段小差≈故障电流，与"区内故障"判定直接对应。模板位置：`references/母差跳闸简报模板.md` §七 7.1。屏显变 220kV Ⅱ母故障（双母单分段）为首个落地案例，其 §七 已分列 ⅠA母小差/ⅠB母小差/Ⅱ母小差。若某套 HDR 未按母段分列小差（如仅给合并值），须：**SGB-750D → 取 RMS CSV 的 `Id_1A`（I母=ⅠA）/ `Id_3A`（III母=ⅠB）通道峰值**，装置 RMS 内部分别计算了三段小差，按支路号背板规则映射（见规则 #26）即可读出，**严禁**标注"未分列ⅠA/ⅠB"。

26. **双母单分段 · SGB-750 支路号直接判定 ⅠA/ⅠB 母段（据南自说明书背板接线规则）**：**18 支路方案**——支路1(母联1) / 支路2(分段) / 支路3(母联2) / **支路4~10 可跨接 Ⅰ、Ⅱ母**（1G=合→ⅠA左短母、2G=合→Ⅱ长母）/ **支路11~18 可跨接 Ⅱ、Ⅲ母**（1G=合→ⅠB右短母、2G=合→Ⅱ长母）。**21 支路方案**——同理，4~12 跨 Ⅰ/Ⅱ，13~21 跨 Ⅱ/Ⅲ。核心规则：**1G 永远接"短母"、2G 永远接"长母(Ⅱ母)"，"短母"是谁（ⅠA or ⅠB）由支路号范围决定**。**落地操作**：① 读 SGB HDR `SettingValue` 获取「支路N名称」确认支路总数（18/21）；② 读 `DigitalStatus` 中每支路的 `支路N_1G/2G刀闸位置` 或 `名称_1G/2G刀闸位置`；③ 按以上 1G/2G→母段 映射直接填入报告 §3.2；④ **严禁再标注"无法区分ⅠA/ⅠB"或臆断"建模差异"**——手册规则已提供确定性判定。**此前屏显变报告中 2N53/4859 标注的「[建模差异]」为规则未查实前的误判，已更正。

## 时序信息融合规则

Subagent 段落按 HDR/Events/RMS 区块组织信息。主 Agent 按以下规则生成时序表：

1. 从各套段落的 **Events信息** 中提取时间信息（events.csv 数据）
2. 从各套段落的 **HDR信息 → TripInfo** 中提取动作时间（标注 `_hdr` 后缀）
3. 相对时间 = 动作时间 - 故障起始时间（故障起始时间在各段落"故障时间"字段中）
4. 列顺序：
   - 双套（220kV+）：厂站1第1套、厂站2第1套、厂站1第2套、厂站2第2套
   - 双端单套：本端、对端
   - 单端单套：本端（仅一列）
5. 动作时序表中不包含电流正负突变行（突变是计算过程数据）
6. HDR 中有但 Events 中无的信号，在时序表中标注 `_hdr` 后缀
7. **时间列使用单点时间**，而非时间段（如"0ms"而非"0~3ms"）

## 越级跳闸分析(2026-06 增量)

### 触发条件

- 主变跳闸事件分析时,需判别是否越级跳下级线路故障
- 需要从主变保护录波 + 故障录波器 或 主变保护录波 + 下级出线录波中识别候选故障线路

### 数据流

```
主变/下级/故障录波器目录
    ↓
[over_trip_analysis.py]
    ↓
├─ auto_select_fault_line (P1/P2/P3 自动选线)
├─ align_cross_device (跨设备时间对齐)
├─ detect_ct_saturation (CT 饱和检测)
├─ Rule 03008 (主变后备越级)
├─ Rule 03009 (CT 饱和致主变差动误动)
└─ Rule 03010 (主变保护未正确动作)
    ↓
越级分析报告.md + JSON
    ↓
[主变跳闸简报模板] 追加 第七章
```

### 依赖 skill

- `transformer-trip-matrix`:提供**主变**跳闸矩阵(由 over_trip_analysis 调用 query_matrix.py)。⚠️ 跳闸矩阵仅主变保护特有;越级分析以其核对主变本体出口,下级出线应跳未跳判定依据保护配置与故障点,不依赖矩阵。
- `latent-faults`:提供 harmonic_analysis.py 复用做 CT 饱和检测

### CLI

```bash
python scripts/over_trip_analysis.py \
  --main-transformer-dir <path> \
  --downstream-dir <path> \
  --fault-recorder-dir <path> \
  --trip-matrix <yaml> \
  --output-dir <path>
```

### 输出文件

`output_dir` 下生成:

- `越级分析报告.md` - 完整 Markdown 报告(7.1~7.6 章节)
- `越级分析结果.json` - 3 条规则执行结果
- `candidate_lines.json` - Top-K 候选故障线路
- `aligned_events.json` - 跨设备对齐后的事件
- `ct_saturation.json` - CT 饱和检测结果
- `推送数据.json` - 保护在线监视系统对接的精简 JSON(嵌入在报告 7.6 章节)
