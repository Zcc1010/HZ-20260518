# 通信通道异常 (communication)

## 类别说明

通信通道异常指保护装置之间的通信链路或通信内容异常，包括GOOSE、SV、光纤通道等。

## 相关信号类型

| 标准标签 | 说明 |
|----------|------|
| GOOSE_ALARM | GOOSE告警 |
| GOOSE_LINK_A | GOOSE A网链路 |
| GOOSE_LINK_B | GOOSE B网链路 |
| SV_ALARM | SV采样值告警 |
| FIBER_LINK | 光纤通道 |
| COMM_LOST | 通信丢失 |

## 典型规则

### Rule 06001: GOOSE通信链路异常

**判定条件**：
1. 检测到GOOSE总告警信号
2. 同时检测到GOOSE接收链路异常信号
3. 时间差在±100ms内

**严重程度**：严重

**证据示例**：
```
装置: 23号
时间: +95ms
信号: ["GOOSE总告警", "链路1GOOSE接收A网链路异常"]
```

**整改建议**：
- 检查GOOSE网络配置
- 确认光纤连接正常
- 检查交换机端口状态

---

### Rule 06002: SV采样值异常

**判定条件**：
1. SV采样值链路中断
2. 采样值质量位异常
3. 采样值数据无效

**严重程度**：危急

**整改建议**：
- 检查合并单元状态
- 检查光纤连接
- 核对采样值配置

---

### Rule 06003: 光纤通道异常

**判定条件**：
1. 光纤通道告警
2. 光功率低于阈值
3. 误码率超标

**严重程度**：危急

**整改建议**：
- 检查光纤连接
- 测试光功率
- 更换故障光模块

---

### Rule 06004: 对侧通信中断

**判定条件**：
1. 纵联保护通道告警
2. 收不到对侧信号
3. 持续时间超过阈值

**严重程度**：严重

**整改建议**：
- 检查两侧通信装置
- 检查通道链路
- 联系对侧变电站

---

### Rule 06005: 通道短时中断

**规则来源**：
- 文档: 继电保护及二次回路隐患缺陷在线辨识技术规范
- 条款: 附录C.7.2

**判定条件**：
1. 通信通道发生中断
2. 中断持续时间 > 20ms
3. 中断后通信自动恢复

**严重程度**：严重

**分析要点**：
- 通道短时中断指通信链路短暂中断后自动恢复，持续时间 > 20ms
- 与通道完全中断（持续 > 2s 或未恢复）的区别
- 可能导致保护装置误动或拒动，影响保护动作一致性

**整改建议**：
1. 检查GOOSE/SV通信链路的光纤连接和接口状态
2. 检查交换机端口配置和状态
3. 检查合并单元和保护装置的通信参数配置
4. 如频繁出现短时中断，应检查通信设备硬件状态
