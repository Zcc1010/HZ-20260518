# 主变保护 Subagent Prompt 模板

## 任务

给定本套主变保护装置的录波文件（hdr + rms.csv + events.csv），按 HDR/Events/RMS 三个区块提取本套装置的关键信息，生成标准化的 Markdown 段落。

## 输入文件

- `*.hdr` — HDR 故障报告文件（XML 格式，可能为 GB18030 或 UTF-8 编码）
- `*.rms.csv` — RMS 统计值文件
- `*.events.csv` — 事件列表文件
- `*.csv` — **原始采样 CSV**（DAT→CSV 输出，含 `时间ms` 列与三相 V/I 通道），用于「故障发展过程数据」区块
- `*.development.json` — **故障发展过程量化数据**（calculate_fault_development.py 产出），可直接引用 stages/key_events/phase_correlation/judgment 填入区块

## 电压等级与接地方式识别（前置条件，必做）

1. **电压等级**：从各侧额定电压（高压侧/中压侧/低压侧，如 220/110/35/10kV）判定。
2. **接地方式**：从 HDR 定值或故障现象判定（大电流=故障相电流大；小电流=35/10kV 低压侧，故障相电压↓电流小、三相基本对称仅零序电压出现）。
3. 将电压等级、接地方式、各侧 CT 配置写入段落开头"**前置条件**"字段，供主 Agent 切换章节（小电流接地下单相接地多为选线/报警而非立即跳闸）。

## 输出格式

严格按照以下 Markdown 格式输出，每个区块的信息都应完整提取，无数据的标注`[无数据]`，有推测规则的标注`[推测为XXX]`。

## 字段提取规则

### 时间格式规范

- **日期**：只写一次（在段落开头故障时间字段中）
- **绝对时间**：格式 `HH:MM:SS.mmm`（六位毫秒，如 `16:33:20.208`）
- **严禁输出相对时间**（如 `0ms`、`15ms`、`约20ms`），所有时间必须为绝对时间 `HH:MM:SS.mmm`
- HDR 中时间是偏移量（如 `0ms`、`11ms`），需转换为绝对时间：
  ```
  绝对时间 = 故障起始时间 + ms偏移量
  ```

### 1. 厂站名和套别

从目录结构或文件名中提取。格式：`{厂站名}_第{X}套`

### 2. 装置型号

从 HDR `<DeviceInfo>` 中提取：
- `装置型号` 或 `装置名称`
- `程序版本` 或 `版本`
- 判断智能站/常规站：根据型号后缀
  - 含 FA/DA/DG → 智能站
  - 其他 → 常规站

### 3. 故障时间

从 HDR `<FaultStartTime>` 提取。格式：`2024-06-28 16:33:20.208`

### 4. HDR设备信息

从 HDR `<DeviceInfo>` 提取以下字段：
- 一次设备名称
- 装置型号
- 程序版本

### 5. TripInfo（保护动作时间序列）

从 HDR `<TripInfo>` 提取，转换为绝对时间：

| HDR字段 | 说明 |
|---------|------|
| time | 偏移量（ms），需转换为绝对时间 |
| name | 动作名称 |
| phase | 相别 |
| value | 1=动作，0=返回 |

### 6. DigitalEvent（数字量事件）

从 HDR `<DigitalEvent>` 提取：

| HDR字段 | 说明 |
|---------|------|
| time | 偏移量（ms），需转换为绝对时间 |
| name | 事件名称 |
| value | 变化描述（如 0→1、1→0） |

### 7. 定值信息

从 HDR `<SettingValue>` 提取关键定值：

| 定值名称 | 说明 | 单位 |
|----------|------|------|
| 差动动作电流定值 | 差动保护动作电流定值 | A |
| 高压侧CT一次额定值 | 高压侧 CT 一次侧额定电流 | A |
| 高压侧CT二次额定值 | 高压侧 CT 二次侧额定电流 | A |
| 中压侧CT一次额定值 | 中压侧 CT 一次侧额定电流 | A |
| 中压侧CT二次额定值 | 中压侧 CT 二次侧额定电流 | A |
| 低压侧CT一次额定值 | 低压侧 CT 一次侧额定电流 | A |
| 低压侧CT二次额定值 | 低压侧 CT 二次侧额定电流 | A |
| PT一次额定值 | PT 一次侧额定电压 | kV |
| 高压侧额定容量 | 高压侧额定容量 | MVA |
| 中压侧额定容量 | 中压侧额定容量 | MVA |
| 低压侧额定容量 | 低压侧额定容量 | MVA |
| 停用重合闸 | 停用重合闸投退（1=停用） | - |

### 8. Events信息

从 `*.events.csv` 提取，保持原始通道名：

| 字段 | 说明 |
|------|------|
| 绝对时间 | 直接使用文件中的时间（时分秒格式） |
| 通道名称 | 原始通道名称 |
| 内容 | 动作/返回 |

### 9. RMS信息

从 `*.rms.csv` 提取：

**各侧电流电压RMS最大值/最小值**（一次值）：
- 高压侧、中压侧、低压侧，全部换算为一次值
- 格式：`{一次值}kA（二次值{值}A）`，无CT变比时 `{值}A [无CT变比]`
- T5变压器（500kV）无中压侧，标注"中压侧—（T5无中压侧）"

**差流**：各侧差动电流值，换算为一次值

**突变信息**：
- 分为"第一次突变"（故障发生时）和"第二次突变"（重合后，如有）
- 列出所有突变幅度超过阈值的通道（涵盖高/中/低压侧各通道）
- 突变发生时间（使用文件中的时间，去掉日期）
- 无第二次突变时不输出第二次分组

### 10. 变压器类型判断

根据目录名或型号判断变压器类型：
- **T2**：220kV变压器，低压侧有总断路器，双分支（低压1分支、低压2分支）
- **T5**：500kV变压器，低压侧无总断路器

### 11. CT变比和PT变比

从 HDR `<SettingValue>` 提取各侧 CT/PT 变比（**不从 CFG 提取**，CFG 中变比通常为 1:1 不准确）：

| 字段名 | 说明 | 单位 |
|--------|------|------|
| 高压侧CT一次额定值 | 高压侧 CT 一次侧额定电流 | A |
| 高压侧CT二次额定值 | 高压侧 CT 二次侧额定电流 | A |
| 中压侧CT一次额定值 | 中压侧 CT 一次侧额定电流 | A |
| 中压侧CT二次额定值 | 中压侧 CT 二次侧额定电流 | A |
| 低压侧CT一次额定值 | 低压侧 CT 一次侧额定电流 | A |
| 低压侧CT二次额定值 | 低压侧 CT 二次侧额定电流 | A |
| PT一次额定值 | PT 一次侧额定电压 | kV |

**CT变比** = CT一次额定值 / CT二次额定值（如 3000/1）

HDR 中无变比字段时，标注 `[无数据]`。

### 12. 一次值换算

各侧电流和电压必须显示一次值，括号内附二次值和变比：

- **电流**：`{一次值}kA（二次值{值}A，CT变比{CT一次}/{CT二次}）`
- **电压**：`{一次值}kV（二次值{值}V，PT变比{PT一次}kV/{PT二次}V）`
- **无变比时**：只显示二次值，标注 `[无CT变比]` 或 `[无PT变比]`

计算公式：
- 电流一次值(kA) = 二次值(A) × CT一次额定值 / CT二次额定值 / 1000
- 电压一次值(kV) = 二次值(V) × PT一次额定值(kV) / PT二次额定值(V)

### 13. FaultInfo（故障信息）

从 HDR `<FaultInfo>` 提取故障时刻的关键电气量，全部换算为一次值：

| 字段名 | 说明 | 显示格式 |
|--------|------|----------|
| 故障相电流 | 故障相电流二次值 | {一次值}kA（二次值{值}A，CT变比{值}/{值}） |
| 最大差动电流 | 差动电流最大值 | 同上 |
| 故障相电压 | 故障相电压二次值 | {一次值}kV（二次值{值}V，PT变比{值}kV/{值}V） |
| 最大零序电流 | 零序电流最大值 | 同电流格式 |

## 编码处理

HDR 可能为 GB18030 或 UTF-8 编码。若读取乱码，尝试另一种编码重新解析。

## 错误处理

- 文件缺失：标注`[本套文件缺失]`
- 字段无数据：标注`[无数据]`
- 有推测规则且可推测：标注`[推测为XXX]`
- 无推测规则：标注`[无数据]`
- T5变压器（500kV）无中压侧：标注"中压侧—（T5无中压侧）"
