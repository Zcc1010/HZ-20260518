# 开关保护（断路器保护）Subagent Prompt 模板

## 任务

给定**断路器保护（开关保护）**装置的录波文件（hdr + rms.csv + events.csv），按 HDR/Events/RMS 三个区块提取本断路器保护的关键动作信息（失灵/死区/三相不一致/充电/重合闸在开关保护），生成标准化的 Markdown 段落。**不判故障类型**——故障类型/差流/距离判据在线路或主变保护段落分析。

## 输入文件

- `*.hdr` — HDR 故障报告文件（XML 格式，可能为 GB18030 或 UTF-8 编码）
- `*.rms.csv` — RMS 统计值文件
- `*.events.csv` — 事件列表文件（含动作/返回事件）

文件位于同一目录下。HDR 文件名示例：`23_00_20240628163320213.hdr`

## 电压等级与接线识别（前置条件，必做）

0. **厂站 / 设备名称 / 中边开关（优先取自目录名）**：
   - 主 Agent 用 `parse_folder_name.py` 解析本装置文件夹，其 `station` / `device_name`（断路器编号）/ `middle_edge` **直接作为段落基线**。
   - **中边判定（按编号）**：`middle_edge=中` 即中断路器（完整串 X2），`边` 即边断路器（X1/X3）；写入段落"前置条件"并用于时序表列头（如"陕州 5042(中)”）。
   - 目录名常不含电压，以 HDR（PT 变比 / 断路器额定电压）补全，注明 `[目录名推断：电压待 HDR 确认]`。
   - HDR 与目录名冲突时以 HDR 为准，标注 `[目录名推断与 HDR 不一致：…]`。

1. **电压等级**：从 PT 变比（500kV/100V、220kV/100V）、断路器额定电压或 HDR 系统参数判定属于 **500（含 750/1000kV）/ 220** kV（开关保护多见于 3/2 高电压等级）。
2. **接线方式**：从目录/定值/一次接线图判定为 **3/2 断路器**（一个半断路器串，每台断路器被两条线路共享）→ 启用本节；常规双母/单母接线一般无独立开关保护分析（N/A）。
3. **设备类型**：**开关保护（断路器保护）**，并标注本断路器所连**关联元件**（线路名称 / 主变侧别）。
4. **关联保护**：同串/同断路器的线路保护或主变保护是否动作、发跳令时刻（用于交叉核对时序）。
5. 将电压等级、接线方式、设备类型、关联元件写入段落开头"**前置条件**"字段，供主 Agent 切换章节。

## 输出格式

严格按照以下 Markdown 格式输出，每个区块的信息都应完整提取，无数据的标注 `[无数据]`，有推测规则的标注 `[推测为XXX]`。

```markdown
## {站名}{断路器编号}_开关保护

**装置型号**: {型号}，{程序版本}，{智能站/常规站}
**制造厂家**: {厂家名称}
**故障时间**: {年}-{月}-{日} {时}:{分}:{秒}.{毫秒}
**关联元件**: {本断路器所连线路名称/主变侧别}

### HDR信息
**设备信息**:
- 一次设备名称: {值}
- 装置型号: {值}
- 程序版本: {值}

**TripInfo（保护动作时间序列）**:
| 绝对时间 | 动作名称 | 相别 | 值 |
|----------|----------|------|-----|
| HH:MM:SS.mmm | {失灵/死区/三相不一致/充电/重合闸…} | {相别} | {1/0} |

**DigitalEvent（数字量事件）**:
| 绝对时间 | 名称 | 变化 |
|----------|------|------|
| HH:MM:SS.mmm | {失灵启动/联跳/重合闸…} | {0→1/1→0} |

**定值信息**:
- 失灵保护定值: {值}（启动条件/延时）
- 死区保护定值: {值}
- 三相不一致定值: {值}（延时）
- 充电保护定值: {值}（若投）
- 重合闸方式(在开关保护): {单重/三重/停用}
- 重合闸时间: {值}s
- CT变比: {CT一次}/{CT二次} 或 [无数据]

### Events信息
| 绝对时间 | 通道名称 | 内容 | 持续时间(ms) |
|----------|----------|------|-------------|
| HH:MM:SS.mmm | {通道名} | 动作/返回 | {值}或空 |

### RMS信息
**电流RMS最大值**: IA/IB/IC（关联线路/主变侧电流，仅作断路器电流是否存在的辅助判据）
**电压RMS最小值**: UA/UB/UC（若有）

### 关联保护交叉核对（必填）
| 项目 | 关联线路/主变保护 | 本开关保护 | 时间差 |
|------|------------------|--------------|--------|
| 发跳令时刻 | {线路/主变保护发跳令绝对时间} | 本开关保护保护启动 | {值}ms |
| 失灵启动 | — | {失灵启动绝对时间} | {值}ms |
| 联跳相邻 | — | {联跳动作绝对时间} | {值}ms |
```

> 数据来源: {hdr文件名} + {rms文件名} + {events文件名}
```

## 字段提取规则

### 时间格式规范

- **日期**：只写一次（在段落开头故障时间字段中）
- **绝对时间**：格式 `HH:MM:SS.mmm`（如 `16:33:20.208`）
- **严禁输出相对时间**（如 `0ms`、`15ms`），所有时间必须为绝对时间 `HH:MM:SS.mmm`
- HDR 中时间是偏移量（如 `0ms`、`11ms`），**必须**转换为绝对时间：`绝对时间 = 故障起始时间 + ms偏移量`

### 1. 站名和断路器编号

从目录结构或文件名中提取。格式：`{站名}{断路器编号}_开关保护`

### 2. 装置型号

从 HDR `<DeviceInfo>` 中提取：装置型号、程序版本；判断智能站/常规站（型号后缀 FA/DA/DG → 智能站）。

### 3. 故障时间

从 HDR `<FaultStartTime>` 提取。格式：`2024-06-28 16:33:20.208`

### 4. 关联元件

从一次接线图/定值/目录结构判定本断路器所连元件（线路名称 或 主变侧别），写入段落。

### 5. TripInfo / DigitalEvent / 定值信息

从 HDR `<TripInfo>` / `<DigitalEvent>` / `<SettingValue>` 提取，转换为绝对时间；重点提取失灵/死区/三相不一致/充电/重合闸（在开关保护）的定值与动作。

**四方（CSC）开关保护软压板提取（重要）**：CSC 系列（CSC-121A、CSC-21A 等）的重合闸软压板不在 `<SettingValue>` 中，在 `<DigitalStatus>` 中以实时状态位呈现——必须提取：
- `<DigitalStatus>` 中的 `停用重合闸`（=1 表示软压板停用）、`重合闸充电完成`（=0 表示未充电）
- `<SettingValue>` 中的控制字（单相重合闸、三相重合闸、停用重合闸）
- `<TripInfo>` 中的实际动作（重合闸动作、低气压闭锁重合闸等）
- 判断重合功能投退 = 控制字 AND DigitalStatus 停用重合闸=0 AND 重合闸充电完成=1（缺一则功能退出）

### 6. Events信息

从 `*.events.csv` 提取，保持原始通道名（如 失灵启动、联跳、重合闸）。**只陈述事实，不做分析**。

### 7. RMS信息

从 `*.rms.csv` 提取关联线路/主变侧电流、电压 RMS（仅作辅助判据，不判故障类型）。

### 8. 关联保护交叉核对（必填）

将本开关保护的保护启动/失灵启动/联跳时刻，与**关联线路/主变保护发跳令时刻**对齐，计算时间差（正常 <20~40ms）。用于主 Agent 在开关保护章节呈现"线路/主变发令 → 断路器执行/失灵/重合"的配合时序。

## 编码处理

HDR 可能为 GB18030 或 UTF-8 编码。若读取乱码，尝试另一种编码重新解析。

## 错误处理

- 文件缺失：标注 `[本套文件缺失]`
- 字段无数据：标注 `[无数据]`
- 有推测规则且可推测：标注 `[推测为XXX]`
- 数值单位不一致：统一换算

## 数据来源标注

在段落末尾标注数据来源文件：

```
> 数据来源: {hdr文件名} + {rms文件名} + {events文件名}
```
