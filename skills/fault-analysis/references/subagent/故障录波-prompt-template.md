# 故障录波 Subagent Prompt 模板

## 任务

给定故障录波器的录波文件（hdr + rms.csv + events.csv），按 HDR/Events/RMS 三个区块提取关键信息，生成标准化的 Markdown 段落。

## 输入文件

- `*.hdr` — HDR 故障报告文件
- `*.rms.csv` — RMS 统计值文件
- `*.events.csv` — 事件列表文件（如有）

## 电压等级与端别识别（前置条件，必做）

1. **电压等级**：从 PT 变比或录波器 HDR 系统参数判定 220/110/66/35/10(6) kV。
2. **端别/站名**：双端配置时本端/对端各一份；单端仅本站一份；35/10kV 可能仅本站一份或缺失。
3. 将电压等级、端别写入段落开头"**前置条件**"字段，供主 Agent 做录波一致性评价。

## 输出格式

严格按照以下 Markdown 格式输出，每个区块的信息都应完整提取，无数据的标注`[无数据]`。

```markdown
## 故障录波_{厂站名}

**录波器型号**: {型号}
**故障时间**: {年}-{月}-{日} {时}:{分}:{秒}.{毫秒}

### HDR信息
**设备信息**:
- 一次设备名称: {值}
- 录波器型号: {值}

**TripInfo（保护动作时间序列）**:
| 绝对时间 | 动作名称 | 相别 | 值 |
|----------|----------|------|-----|
| HH:MM:SS.mmm | {动作名} | {相别} | {1/0} |

**FaultInfo（故障信息）**:
- 故障相电流: {一次值}kA（二次值{值}A，CT变比{值}/{值}）或 {值}A [无CT变比]
- 故障相电压: {一次值}kV（二次值{值}V，PT变比{值}kV/{值}V）或 {值}V [无PT变比]

**定值信息**:
- 故障测距: {值}km
- 故障相别: {相别}
- CT变比: {CT一次额定值}/{CT二次额定值} 或 [无数据]
- PT变比: {PT一次额定值}kV 或 [无数据]

### Events信息
| 绝对时间 | 通道名称 | 内容 |
|----------|----------|------|
| HH:MM:SS.mmm | {通道名} | 动作/返回 |

### RMS信息
**电流RMS最大值**（一次值）: IA: {一次值}kA（二次值{值}A）, IB: ..., IC: ..., 3I0: ...
（无CT变比时: IA: {值}A [无CT变比]）
**电压RMS最小值**（一次值）: UA: {一次值}kV（二次值{值}V）, UB: ..., UC: ..., 3U0: ...
（无PT变比时: UA: {值}V [无PT变比]）

**电流电压突变**:
**第一次突变**（故障发生时）:
- {通道名}正突变: +{值}A @ HH:MM:SS.mmm
- {通道名}负突变: -{值}A @ HH:MM:SS.mmm
- ...（列出所有突变幅度超过阈值的通道）

**第二次突变**（重合后，如有）:
- {通道名}正突变: +{值}A @ HH:MM:SS.mmm
- ...（无第二次突变时不输出此分组）

> 数据来源: {hdr文件名} + {rms文件名} + {events文件名}
```

## 字段提取规则

### 时间格式规范

- **日期**：只写一次（在段落开头故障时间字段中）
- **绝对时间**：格式 `HH:MM:SS.mmm`（六位毫秒，如 `16:33:20.208`）
- **严禁输出相对时间**（如 `0ms`、`15ms`、`约20ms`），所有时间必须为绝对时间 `HH:MM:SS.mmm`
- HDR 中时间是偏移量，需转换为绝对时间：
  ```
  绝对时间 = 故障起始时间 + ms偏移量
  ```

### 1. 厂站名

从目录结构或文件名中提取。格式：`故障录波_{厂站名}`

### 2. 录波器型号

从 HDR `<DeviceInfo>` 提取。

### 3. 故障时间

从 HDR `<FaultStartTime>` 提取。格式：`2024-06-28 16:33:20.208`

### 4. HDR设备信息

从 HDR `<DeviceInfo>` 提取：
- 一次设备名称
- 录波器型号

### 5. TripInfo（保护动作时间序列）

从 HDR `<TripInfo>` 提取，转换为绝对时间：

| HDR字段 | 说明 |
|---------|------|
| time | 偏移量（ms），需转换为绝对时间 |
| name | 动作名称 |
| phase | 相别 |
| value | 1=动作，0=返回 |

### 6. 定值信息

从 HDR `<FaultInfo>` 或 `<TripInfo>` 中提取：
- 故障测距
- 故障相别

### 7. Events信息

从 `*.events.csv` 提取，保持原始通道名：

| 字段 | 说明 |
|------|------|
| 绝对时间 | 直接使用文件中的时间（时分秒格式） |
| 通道名称 | 原始通道名称 |
| 内容 | 动作/返回 |

### 8. RMS信息

从 `*.rms.csv` 提取：

**电流RMS最大值**（一次值）：
- IA、IB、IC、3I0电流最大值，换算为一次值
- 格式：`{一次值}kA（二次值{值}A）`

**电压最小值**（一次值）：
- UA、UB、UC、3U0电压最小值，换算为一次值
- 格式：`{一次值}kV（二次值{值}V）`

**突变信息**：
- 分为"第一次突变"（故障发生时）和"第二次突变"（重合后，如有）
- 列出所有突变幅度超过阈值的通道
- 突变发生时间（使用文件中的时间，去掉日期）
- 无第二次突变时不输出第二次分组

## 编码处理

HDR 可能为 GB18030 或 UTF-8 编码。若读取乱码，尝试另一种编码重新解析。

## CT变比和PT变比

从 HDR `<SettingValue>` 提取 CT/PT 变比（**不从 CFG 提取**，CFG 中变比通常为 1:1 不准确）：

| 字段名 | 说明 | 单位 |
|--------|------|------|
| CT一次额定值 | CT 一次侧额定电流 | A |
| CT二次额定值 | CT 二次侧额定电流（通常 1A 或 5A） | A |
| PT一次额定值 | PT 一次侧额定电压 | kV |

**CT变比** = CT一次额定值 / CT二次额定值（如 3000/1）

HDR 中无变比字段时，标注 `[无数据]`。

## 一次值换算

电流和电压必须显示一次值，括号内附二次值和变比：

- **电流**：`{一次值}kA（二次值{值}A，CT变比{CT一次}/{CT二次}）`
- **电压**：`{一次值}kV（二次值{值}V，PT变比{PT一次}kV/{PT二次}V）`
- **无变比时**：只显示二次值，标注 `[无CT变比]` 或 `[无PT变比]`

计算公式：
- 电流一次值(kA) = 二次值(A) × CT一次额定值 / CT二次额定值 / 1000
- 电压一次值(kV) = 二次值(V) × PT一次额定值(kV) / PT二次额定值(V)

## FaultInfo（故障信息）

从 HDR `<FaultInfo>` 提取故障时刻的关键电气量，全部换算为一次值：

| 字段名 | 说明 | 显示格式 |
|--------|------|----------|
| 故障相电流 | 故障相电流二次值 | {一次值}kA（二次值{值}A，CT变比{值}/{值}） |
| 故障相电压 | 故障相电压二次值 | {一次值}kV（二次值{值}V，PT变比{值}kV/{值}V） |

## 错误处理

- 文件缺失：标注`[录波文件缺失]`
- 字段无数据：标注`[无数据]`
