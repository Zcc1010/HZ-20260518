# 线路保护 Subagent Prompt 模板

## 任务

给定本套保护装置的录波文件（hdr + rms.csv + events.csv），按 HDR/Events/RMS 三个区块提取本套装置的关键信息，生成标准化的 Markdown 段落。

## 输入文件

- `*.hdr` — HDR 故障报告文件（XML 格式，**编码因厂家而异：南瑞继保 PCS/PRS 多为 GB18030；国电南瑞 NSR 多为 UTF-8 带 BOM**；读取乱码时换另一种编码重解）
- `*.rms.csv` — RMS 统计值文件
- `*.events.csv` — 事件列表文件（含动作/返回事件）
- `*.csv` — **原始采样 CSV**（DAT→CSV 输出，含 `时间ms` 列与三相 V/I 通道），用于「故障发展过程数据」区块
- `*.development.json` — **故障发展过程量化数据**（calculate_fault_development.py 产出），可直接引用 stages/key_events/phase_correlation/judgment 

文件位于同一目录下。HDR 文件名示例：`23_00_20240628163320213.hdr`

## 电压等级与接地方式识别（前置条件，必做）

0. **厂站 / 设备名称 / 保护套别（优先取自目录名）**：
   - 若主 Agent 已用 `parse_folder_name.py` 解析过本装置所在文件夹，其输出的 `station` / `device_name` / `set_cn` / `voltage_kv` / `model` / `middle_edge` **直接作为段落"前置条件"的基线**，无需从 HDR 反推。
   - 目录名无法确定的项（如断路器目录常无电压）以 HDR（PT 变比 / 线路额定电压 / 定值）补全，并在段落注明 `[目录名推断：电压待 HDR 确认]`。
   - **HDR 与目录名冲突时以 HDR 为准**，并标注 `[目录名推断与 HDR 不一致：…]`（例：目录名"第一套"但 HDR 实例为第 2 套）。
   - 3/2 接线：目录名解析出的 `middle_edge`（X2=中 / X1、X3=边）用于标注本断路器是"中断路器"还是"边断路器"，供 §6.7 开关保护分析对齐。

1. **电压等级**：从 PT 变比（500kV/100V、220kV/100V、110kV/100V、35kV/100V、10kV/100V）、线路额定电压或 HDR 系统参数判定属于 **500（含 750/1000kV）/ 220 / 110 / 66 / 35 / 10(6) kV**。
2. **套别（配置数）**：
   - 500kV+ / 220kV+ → 双套（每端 2 套，共 4 套）。
   - 110kV 及以下 → 单套（双端每端 1 套 = 双端单套；单端 = 单端单套）。
   - **单端单套"装置内双套电流"红线**：若本装置记录两套电流（保护电流1/2），按 `电压等级保护配置知识.md §6.1` 判为装置内本地量，**绝非本侧/对侧**，禁止当作纵差差流。
3. **接地方式**：从 HDR 定值（"小电流接地""消弧线圈""经电阻""直接接地"）或故障现象判定（大电流=故障相电流大且不对称；小电流=故障相电压↓电流小、三相基本对称仅零序电压出现）。
4. **接线方式与开关保护（500kV+ 关键）**：若接线为 **3/2 断路器**（500kV+ 为主，220kV 部分站亦用），则**重合闸逻辑在断路器保护（开关保护）中实现、非本线路保护**，且须独立分析**断路器失灵/死区/三相不一致/充电**动作。若事故目录含 `断路器保护`/`开关保护` 录波文件，本线路段落须注明"开关保护分析见其独立段落/简报 §6.7"，并交叉核对"线路保护发跳令 t → 断路器保护执行跳闸/失灵/重合"的时序（详见 `电压等级保护配置知识.md §6.5`）。3/2 接线**无常规母差**，母差章节整篇 N/A。
5. 将电压等级、套别、接地方式、接线方式写入段落开头"**前置条件**"字段，供主 Agent 切换章节。

## 字段提取规则

### 时间格式规范

- **日期**：只写一次（在段落开头故障时间字段中）
- **绝对时间**：格式 `HH:MM:SS.mmm`（如 `16:33:20.208`）
- **严禁输出相对时间**（如 `0ms`、`15ms`、`约20ms`），所有时间必须为绝对时间 `HH:MM:SS.mmm`
- HDR 中时间是偏移量（如 `0ms`、`11ms`），**必须**转换为绝对时间：
  ```
  绝对时间 = 故障起始时间 + ms偏移量
  例：故障起始 16:33:20.213 + 15ms = 16:33:20.228
  ```

### 1. 厂站名和套别

从目录结构或文件名中提取。格式：`{厂站名}_第{X}套`

### 2. 装置型号

**设备型号 / 版本号的权威来源优先级（HDR DeviceInfo > 录波文件名 > CFG 第1行）：**

- **HDR `<DeviceInfo>` 中的装置型号是第一手权威来源**——HDR 是保护装置自身生成的故障报告，Device Info 中的型号为装置完整型号（如 `CSC-121A-DG-GCN`）。当 .zwav 文件名与 HDR DeviceInfo 不一致时，**以 HDR 为准**。
- **录波文件名（.zwav，GBK 解码后）是第二来源**——录波文件由保护装置自身生成并命名，含装置型号（如 `CSC-21A-DG-GCN`），但可能被截断（如 `CSC-21A`），以 HDR 完整体为准。
- **CFG 第1行（COMTRADE 标准头 `站名,装置ID,版本年`）是第二来源**，必须读取：
  - **国电南瑞 NSR（UTF-8+BOM 编码）**：CFG 第1行形如 `国网六统一版本,NSR-303A-DA-G-R,1999`：
    - field1 = **版本/平台标识**（如 `国网六统一版本`，对应 PCS 的 V3.60 数字版本，**这就是 NSR 的"版本号"**）；
    - field2 = **完整装置型号**（如 `NSR-303A-DA-G-R`，**含 `-DA`→智能站**；注意 HDR 的 `被保护设备=NSR303` 是被截断的，以 CFG 第1行 field2 的完整体为准）；
    - field3 = CFG 版本年（如 `1999`）。
    - ⚠️ NSR 的 HDR **无 `<DeviceInfo>` 块、也无"版本号"字段**——型号与版本**只在 CFG 第1行**，漏读 CFG 就会误判"NSR 基础信息缺失"。
  - **南瑞继保 PCS/PRS（GB18030 编码）**：CFG 第1行形如 `南瑞继保,超高压输电线路成套保护装置,1999`：
    - field1 = 制造厂家（`南瑞继保`）、field2 = 装置描述（非数字型号）、field3 = 版本年；
    - **精确数字型号（如 `PCS-931A-DA-G-R`）来自 HDR `<DeviceInfo><name>被保护设备</name>` 或文件夹名**，CFG 第1行只给厂家+描述。
- **站型判定（三重证据，任一满足即智能站）**：
  - 型号后缀含 **FA/DA/DG → 智能站**（NSR 看 CFG 第1行 field2 完整型号 `NSR-303A-DA-G-R` 的 `-DA`，**勿信 HDR 截断的 `NSR303`**）；
  - 或 HDR `<DigitalEvent>` 出现 `SV接收软压板` / `*GOOSE链路中断` / `智能终端` / `合并单元(MU)` 等过程层数字量 → 智能站；
  - 否则常规站。
- **厂站名**：NSR HDR `厂站名=STATION`、CFG 第1行 field1=`国网六统一版本` 都是**占位/平台标识，非真站名**；真站名仍依文件夹名 / IED 代码 / 调度命名。

### 3. 故障时间

从 HDR `<FaultStartTime>` 提取。格式：`2024-06-28 16:33:20.208`

### 4. HDR/CFG 设备信息

**按厂家取数（见 §2 结构说明）**：
- **首先读取 CFG 第1行（COMTRADE 头 `站名,装置ID,版本年`）取型号+版本**：NSR 的 field1=版本平台(`国网六统一版本`)、field2=完整型号(`NSR-303A-DA-G-R`，含-DA)；PCS 的 field1=厂家、field2=装置描述，精确型号另取 HDR/文件夹。
- 南瑞继保 PCS/PRS：从 HDR `<DeviceInfo>` 提取 `一次设备名称` / `装置型号` / `程序版本`(=`<name>版本号</name>`的值，如 `V3.60`)。
- 国电南瑞 NSR：HDR **`<DeviceInfo>` 不存在**（设备信息散在 `<SettingValue>`：被保护设备/厂站名/间隔开关名称）；**型号与版本号在 CFG 第1行**（非 HDR），务必读 CFG 第1行而非标"HDR未记录"。
- 编码：PCS 为 GB18030、NSR 为 UTF-8（带 BOM）。若读取乱码，按对端编码重解（见本模板末尾编码提示）。
- **PT/CT 变比**：NSR CFG 通道比例常载真实一次/二次值（如 `220000/100`→PT变比2200、`2500/5`→CT变比500）；PCS CFG 通道比例多为归一化 `1/1`，PT/CT 须改从 HDR `CT一次/二次额定值` 取。同线路多套共用同一 PT，可互相印证。

### 5. TripInfo（保护动作时间序列）

从 HDR `<TripInfo>` 提取，转换为绝对时间：

| HDR字段 | 说明 |
|---------|------|
| time | 偏移量（ms），需转换为绝对时间 |
| name | 动作名称 |
| phase | 相别（如ABC、A相、无） |
| value | 1=动作，0=返回 |

**转换方法**：
```
绝对时间 = 故障起始时间 + time偏移
例：故障起始 16:33:20.208，HDR中 time=107ms
    绝对时间 = 16:33:20.208 + 107ms = 16:33:20.315
```

### 9. RMS信息

从 `*.rms.csv` 提取：

**电流RMS最大值**：
- IA、IB、IC、3I0（零序电流）的RMS最大值，全部换算为一次值
- 格式：`{一次值}kA（二次值{值}A）`，无CT变比时 `IA: {值}A [无CT变比]`

**电压RMS最小值**：
- UA、UB、UC、3U0的RMS最小值，全部换算为一次值
- 格式：`{一次值}kV（二次值{值}V）`，无PT变比时 `UA: {值}V [无PT变比]`

**电流电压突变**（按故障次数分组）：
- 分为"第一次突变"（故障发生时）和"第二次突变"（重合后，如有）
- 列出所有突变幅度超过阈值的通道（不限 IA/UA/3U0）
- 突变发生时间（使用文件中的时间，去掉日期）
- 无第二次突变时不输出第二次分组

### 10. CT变比和PT变比

从 HDR `<SettingValue>` 提取 CT/PT 变比（**不从 CFG 提取**，CFG 中变比通常为 1:1 不准确）：

| 字段名 | 说明 | 单位 |
|--------|------|------|
| CT一次额定值 | CT 一次侧额定电流 | A |
| CT二次额定值 | CT 二次侧额定电流（通常 1A 或 5A） | A |
| PT一次额定值 | PT 一次侧额定电压 | kV |

**CT变比** = CT一次额定值 / CT二次额定值（如 3000/1）
**PT变比** = PT一次额定值 / PT二次额定值

HDR 中无 CT/PT 变比字段时，标注 `[无数据]`。

### 11. 一次值换算

电流和电压必须显示一次值，括号内附二次值和变比：

- **电流**：`{一次值}kA（二次值{值}A，CT变比{CT一次}/{CT二次}）`
- **电压**：`{一次值}kV（二次值{值}V，PT变比{PT一次}kV/{PT二次}V）`
- **无变比时**：只显示二次值，标注 `[无CT变比]` 或 `[无PT变比]`

计算公式：
- 电流一次值(kA) = 二次值(A) × CT一次额定值 / CT二次额定值 / 1000
- 电压一次值(kV) = 二次值(V) × PT一次额定值(kV) / PT二次额定值(V)

### 12. FaultInfo（故障信息）

从 HDR `<FaultInfo>` 提取故障时刻的关键电气量，全部换算为一次值：

| 字段名 | 说明 | 显示格式 |
|--------|------|----------|
| 故障相电流 | 故障相电流二次值 | {一次值}kA（二次值{值}A，CT变比{值}/{值}） |
| 最大差动电流 | 差动电流最大值 | 同上 |
| 故障相电压 | 故障相电压二次值 | {一次值}kV（二次值{值}V，PT变比{值}kV/{值}V） |
| 最大零序电流 | 零序电流最大值 | 同电流格式 |

### 13. 距离I段不动作检查项（必填，4套均无距离I段动作时强化输出）

**触发场景**: 四套装置Events信息中均无"接地距离Ⅰ段动作"信号。

**13.1 距离保护Ⅰ段有效控制字**
- 来源：HDR `<DigitalStatus>` 查找 `距离保护Ⅰ段有效` 或 `距离Ⅰ段投入` 字段
- 值=0 → I段被退出（人为配置）
- 值=1 → I段已投入，需进一步判别
- 无字段 → 标注 `[无数据]`，默认按"退出"处理并写明依据

**13.2 距离I段保护范围核算**

保护范围(km) ≈ 接地距离Ⅰ段定值(Ω) ÷ Z1(Ω/km)

- 来源1：HDR `<SettingValue>` 的 `接地距离Ⅰ段定值` (阻抗定值，单位Ω)
- 来源2：HDR `<SettingValue>` 或扩展定值单的 `线路正序阻抗 Z1` (Ω/km) 或 `线路总长度+总阻抗` 反推
- 若 Z1 字段缺失而只有 L(km) 和 Z1_总(Ω)：Z1 = Z1_总 / L

**13.3 不动作原因判别规则**

| 故障点 vs I段范围 | 控制字=0(退出) | 控制字=1(投入) |
|---------|----------------|-----------------|
| 故障点 > I段范围 | 正确不动作 | 正确不动作(物理上不会动作) |
| 故障点 ≤ I段范围 | 正确不动作(主动退出) | **应动未动，待核实** |

**13.4 段落输出格式**

必须输出"距离I段不动作原因"区块（即使控制字=0也要输出，便于审计追溯）：

```
### 距离I段不动作原因（必填）

| 判别项 | 值 | 说明 |
|--------|----|------|
| 控制字(距离保护Ⅰ段有效) | 0 | 退出 |
| Ⅰ段定值(阻抗) | 0.25 Ω | - |
| Ⅰ段时间定值 | 0.00 s | - |
| Ⅰ段保护范围(估算) | 0.7 km | Z1=0.358 Ω/km |
| 本套测距(HDR) | 2.33 km | - |
| 故障点 vs Ⅰ段范围 | 在范围外 | (2.33 > 0.7) |
| 结论 | 正确不动作 | 即使投入也不应动作 |
```

**13.5 注意事项**

- 距离I段未动 ≠ 缺陷，必须按本节3步骤判别后才可下结论
- 控制字=0 时，不动作 100% 正确；不可将此误判为"未投保护"
- 控制字=1 + 故障点 > I段范围 时，仍属正确（物理不可达）
- 控制字=1 + 故障点 ≤ I段范围 时，**必须**标 `[待核实: 距离I段应动未动]`，触发隐患规则

## 编码处理

HDR 可能为 GB18030 或 UTF-8 编码。若读取乱码，尝试另一种编码重新解析。

## 错误处理

- 文件缺失：标注`[本套文件缺失]`
- 字段无数据：标注`[无数据]`
- 有推测规则且可推测：标注`[推测为XXX]`
- 无推测规则：标注`[无数据]`
- 数值单位不一致：统一换算（如 Ω 和 ohm 等效）
