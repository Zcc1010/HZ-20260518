# 目录命名解析（厂站 / 设备名称 / 保护套别 推断）

> 配套脚本：`scripts/parse_folder_name.py`
> 适用阶段：电网故障智能分析 工作流程 **第 0 步（配置识别前置）**。录波文件夹通常以
> “事故文件夹 / 厂站 / 设备描述 /” 层级组织，**设备描述目录名即编码了厂站外的全部关键信息**。
> 本文件把“从目录名推断”系统化为可复用规则，避免每回分析都靠人工肉眼分辨。

## 1. 为什么需要目录名解析

电力录波归档命名虽不强制统一，但现场普遍遵循“能看出是哪套装置”的约定。一条完整路径往往自带：

| 维度 | 常见写法 | 示例 |
|------|----------|------|
| 电压等级 | `500kV` `220kV` `35kV` | `500kV陕嘉Ⅱ回线…` |
| 设备类型 | 线路/光差/纵差、断路器/开关、母差、主变、电容器… | `…线路保护…` `…断路器…` |
| 设备名称 | 线路名、断路器编号 | `陕嘉Ⅱ回线` `5042` |
| 保护套别 | 第一套/第二套（或 A/B 后缀） | `…第一套保护…` |
| 装置型号 | 制造厂型号前缀 | `PCS931AG` `PRS753NA` `PCS921A` |
| 3/2 中边 | 断路器编号末位（完整串 X2=中、X1/X3=边） | `5042`→中 `5043`→边 |

与既有原则一致：**HDR 内容 > 目录名称 > 用户确认**。目录名是“首选快速推断”，
最终仍以 HDR 校验；凡目录名无法确定的项，一律标注 `待核实`，绝不臆造。

## 2. 脚本用法

```bash
# 解析一个事故文件夹（自动发现所有设备子目录）
python <skill>/scripts/parse_folder_name.py "事故文件夹/"

# 解析多个路径（文件或目录均可，自动判别设备层）
python <skill>/scripts/parse_folder_name.py "事故文件夹/厂站A" "事故文件夹/厂站B"

# 单个录波文件 → 取其父目录为设备文件夹
python <skill>/scripts/parse_folder_name.py "…/5042断路器PCS921A/P_B5042X_…_f.zip"

# 机器可读输出（供主 Agent / 自动化消费）
python <skill>/scripts/parse_folder_name.py "事故文件夹/" --json

# 显式指定厂站（覆盖推断）/ 指定事故根目录
python <skill>/scripts/parse_folder_name.py "…/" --station 陕州 --root "事故文件夹/"
```

输出字段（JSON / 表格一一对应）：

| 字段 | 含义 |
|------|------|
| `station` / `station_confidence` | 厂站（取设备文件夹的父目录）；`high`=父目录非容器名，`low`/`override`=容器层或命令行指定 |
| `device_name` | 设备名称（线路名 / 断路器编号 / 主变号 / 母线段） |
| `device_type` / `device_type_cn` | 标准化设备类型（line/breaker/busbar/transformer/distribution → 中文） |
| `set_no` / `set_cn` | 保护套别（第1/2/3套），断路器保护通常记“单套”以中边区分 |
| `voltage_kv` | 电压等级（目录名含 `xxkV` 时提取） |
| `model` | 装置型号（制造厂前缀正则提取） |
| `breaker_no` / `middle_edge` | 断路器编号 / 中边（仅断路器类型、编号完整时） |
| `needs_review` | 待核实清单（电压缺失、父目录为容器、中边末位非 1/2/3 等） |

## 3. 真实案例（500kV 陕嘉Ⅱ线）

│   ├── 500kV陕嘉Ⅱ回线第二套保护PRS753NA/      → 厂站=嘉和 设备=陕嘉Ⅱ回线 类型=线路 套别=第二套 型号=PRS753NA

## 4. 解析规则（核心正则与优先级）

### 4.1 设备类型（关键词优先级，**顺序即优先级**）

| 优先级 | 类型 | 触发关键词 |
|--------|------|------------|
| 1 | 开关保护（断路器保护） | 断路器、开关、失灵、breaker |
| 2 | 母差保护 | 母差、母线、busbar、bus |
| 3 | 配电设备 | 电容器、电容、电抗器、电抗、站用变、接地变、所用变、配电 |
| 4 | 主变保护 | 主变、变压器（**不用裸“变”**，否则 站用变/接地变 会被误判） |
| 5 | 线路保护（兜底） | 光差、纵差、分相电流差动、线路、回线、线 |

> 配电须排在 主变 之前；主变仅认“主变/变压器”，避免 站用变/接地变 含“变”而误归主变。

### 4.2 保护套别

- 显式：`第一套/第1套/一套`→1，`第二套/第2套/二套`→2，`第三/四套`→3/4；罗马数字 `Ⅰ/Ⅱ/Ⅲ` 同 1/2/3。
- 线路 A/B 后缀（如 `L5011A`=第1套、`L5011B`=第2套）属文件名层，本脚本按目录名的“第X套”判定；
  若目录名无套别词，线路保护 `set_cn` 留空（由 HDR 证实），断路器保护记“单套”以中边区分。

### 4.3 装置型号（制造厂前缀）

正则：`(?:PCS|PRS|CSC|NSR|RCS|BP|WBC|ISA|SAC|SEL|GE|ABB|NR|GUID)\-?\d{1,4}[\-A-Za-z]*`
（大小写不敏感）。示例命中：`PCS931AG`、`PRS753NA`、`PCS921A`、`BP-2C`、`RCS-915`。
仅取“制造厂前缀型号”，**不**会把 `L5011A`、`5042` 这类设备编号误当型号。

### 4.4 设备名称提取

1. 先按设备类型取：断路器→取编号（`(\d{3,4})`，已剔除型号串中的数字避免误吞 `921`）；
   其余类型→先做类型/电压/套别/型号词清理，保留剩余描述（如 `陕嘉Ⅱ回线`、`崇本线`、`Ⅰ母`、`2号主变`、`电容器组`）。
2. 清理**不删**设备名称词（主变、电容器、站用变、电抗器等保留在名称里），只删类型【关键词】与套别/电压/型号/保护/装置等通用词。
3. 清理后为空 → 先剥型号/保护/装置再试，仍空则回退原目录名并标 `待核实`。

### 4.5 3/2 接线 中边开关判定（按断路器编号）

> 用户约定：**对完整串，X2=中断路器，X1/X3=边断路器**。

- 断路器编号末位 `2` → **中**；末位 `1` 或 `3` → **边**；末位为其他 → `middle_edge=None` 并标 `待核实`。
- 仅当 `device_type=breaker` 且编号完整（3~4 位）时输出中边；不完整串（如仅 1~2 台）
  该规则未必适用，须结合 HDR/接线图确认。
- 该规则与 HDR 重合闸时序可互验：边开关先合、中开关后合（约 500ms 间隔）。

### 4.6 厂站推断与置信度

- 设备文件夹的**直接父目录**即为厂站候选。
- 若父目录名属于“容器/事故层”关键词（`保护录波`、`故障录波`、`录波`、`事故`、`output`、
  `提取`、`extracted`、`分析`、`波形`、`data`、`files`、`zip`），则厂站无法从路径确定 →
  `station_confidence=low`，标注“厂站需从 HDR 或用户确认”，可用 `--station` 显式指定。
- 典型三层（`事故/厂站/设备`）下父目录即厂站，`confidence=high`。

## 5. 与既有流程的衔接

1. **第 0 步前置**：先跑 `parse_folder_name.py` 得到 `{厂站, 设备名称, 类型, 套别, 电压, 型号, 中边}` 表；
   再结合 HDR 内容做“电压等级开关”判定（见 `电压等级保护配置知识.md` §9）。
2. **Subagent 段落**：将目录推断的厂站/设备/套别作为段落“前置条件”的输入；HDR 与目录名冲突时，
   以 HDR 为准并在简报标注 `[目录名推断与 HDR 不一致：…]`。
3. **主 Agent 简报**：保护配置表（第二节/第三节）的“厂站/套别/设备/型号”列直接采用目录推断结果，
   `needs_review` 中的待核实项转为简报 `[待核实]` 注记。

## 6. 限制与注意

- 目录名是“约定俗成”，非强制标准；个别单位命名可能不含电压/套别，此时靠 HDR 补。
- 型号前缀表为常见厂家的典型前缀，遇到新厂家型号未命中时 `model=None` 并标 `待核实`。
- 中边判定依赖“完整串 X2=中”约定，不适用于不完整串或非 3/2 接线；务必以 HDR/接线图复核。
