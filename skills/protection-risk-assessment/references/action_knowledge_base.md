# 处置建议知识库

> 基于规则维度生成 1-3 条**可操作**建议。
> 建议必须具体到"做什么"、"参照什么"、"时间窗口"——避免空泛"加强巡视"。

## 一、建议模板规则

**原则：**
1. 每条建议开头动词明确（立即 / 安排 / 列入 / 排查）。
2. 给出操作对象（装置 + 部件）。
3. 给出参照文件或调度单编号（如果有）。
4. 给出时间窗口（≤4h / ≤24h / ≤7d / 列入计划）。

## 二、按规则维度建议模板

### A1 CPU/板卡严重故障

| 顺序 | 建议 |
|---|---|
| 1 | 立即通知调度，按调度指令考虑退出该保护或投运备用保护。 |
| 2 | 启动紧急消缺流程，按厂家技术说明书更换指定版本板卡。 |
| 3 | 排查本站同批次装置，按反事故措施登记造册。 |

### A2 装置自检告警持续

| 顺序 | 建议 |
|---|---|
| 1 | 立即安排现场人员检查装置显示屏告警代码。 |
| 2 | 调取最近一次检修记录，比对告警变化趋势。 |
| 3 | 若 24h 内未自行复归，列入本周消缺计划。 |

### A3 参数采集异常

| 顺序 | 建议 |
|---|---|
| 1 | 检查保护管理机/通讯网关链路是否正常。 |
| 2 | 比对通讯中断时间窗口与现场检修/调试记录。 |
| 3 | 通讯恢复后核对装置参数与定值区一致性。 |

### A4 检修超期/缺陷超时

| 顺序 | 建议 |
|---|---|
| 1 | 立即核实缺陷/检修超期精确时长，按规程分级上报。 |
| 2 | 结合一次停电计划安排保护装置首检/复检。 |
| 3 | 暂时降级或退出超期装置所辖保护段（按调度指令）。 |

### A5 家族性缺陷

| 顺序 | 建议 |
|---|---|
| 1 | 按反事故措施文件编号锁定批次。 |
| 2 | 立即安排停电窗口更换指定部件。 |
| 3 | 在家族性缺陷全网排查完成前，加强相关装置巡视。 |

### B1 通道异常

| 顺序 | 建议 |
|---|---|
| 1 | 立即检查光纤通道一/二收发光功率。 |
| 2 | 检查保护室配线架、对侧光端机状态。 |
| 3 | 联系通道运维班组现场测试。 |

### B2 纵联保护闭锁/闭锁主保护

| 顺序 | 建议 |
|---|---|
| 1 | 立即核对闭锁信号来源（通道/对侧/自身）。 |
| 2 | 复归信号并通知调度。 |
| 3 | 闭锁持续 ≥ 1h 启动消缺流程。 |

### B3 通讯状态异常

| 顺序 | 建议 |
|---|---|
| 1 | 检查保信子站/主站通讯状态。 |
| 2 | 必要时切换至备用数据通道。 |
| 3 | 通讯恢复前暂停对此装置的远程定值修改。 |

### D1 定值区号不符

| 顺序 | 建议 |
|---|---|
| 1 | 立即核对调度定值单（区号 + 定值明细）。 |
| 2 | 按调度指令切换定值区，核对模拟量中"定值区号"。 |
| 3 | 在 1h 内完成切换并通知调度校核。 |

### D2 关键定值漂移

| 顺序 | 建议 |
|---|---|
| 1 | 调取最近一次调度定值单进行比对。 |
| 2 | 核对修改记录与工作票。 |
| 3 | 若无合法变更记录，按调度指令回退。 |

### D3 运行方式不匹配

| 顺序 | 建议 |
|---|---|
| 1 | 核对运行方式通知单与当前定值单。 |
| 2 | 按规定停用/启用相关保护功能（如重合闸、纵联）。 |
| 3 | 完成切换后在运行日志留底。 |

## 三、行业通用建议（兜底）

| 类型 | 建议 |
|---|---|
| 任何危急 | 通知调度；考虑方式调整；启动紧急消缺。 |
| 任何严重 | 列入近 7 天消缺计划；加强定期巡视。 |
| 任何一般 | 列入月度检修计划；按工作票执行。 |
| 任何提示 | 列入年度反措通知/计划检修日志。 |
