# 风险简报模板规范

> 核心交付物的格式约束。简报必须**简洁、聚焦、可追溯**。

## 一、四级视觉化标识

| 等级 | Emoji | 颜色 | 文字 |
|---|---|---|---|
| 危急 | 🔴 | 红色 | 危急 |
| 严重 | 🟠 | 橙色 | 严重 |
| 一般 | 🟡 | 黄色 | 一般 |
| 提示 | ⚪ | 灰色 | 提示 |

## 二、必含字段

每份简报必须包含以下字段，按顺序：

1. **标题行** —— `[Emoji] 风险简报`
2. **风险等级** —— `🔴 危急` / `🟠 严重` / ...
3. **影响范围** —— `厂站 > 间隔/线路 > 保护装置`
4. **风险概述** —— 1-2 句精炼概括风险点
5. **核心建议** —— 1-3 条可操作建议
6. **推理追溯** —— 点击展开（活页机制）

## 三、字段详细规范

### 风险等级字段

```
● 风险等级：🔴 危急
```

- 必须 Emoji + 文字，**禁止只写文字**
- 危急情况可附加**【预警】**前缀以触发明显视觉警示

### 影响范围字段

```
● 影响范围：500kV 皋城变 → 皋文 5325 线 → 第一套 PCS-931SA-G
```

- 用 `→` 分级（不是 `/` 也不是 `-`）
- 三级：厂站 / 间隔/线路 / 保护装置
- 多装置风险：分行列出每套装置

### 风险概述字段

```
● 风险概述：XX线第一套线路保护 CPU 板卡严重故障告警未复归，
           且该型号家族性反措未执行，在线路重载方式下存在主保护拒动风险。
```

- 单句 ≤ 60 字
- 包含：核心异常 + 时间窗口 + 可能后果

### 核心建议字段

```
● 核心建议：
  1. 立即通知调度，按调度指令退出该保护。
  2. 启动紧急消缺流程，参照反措文件编号 XX 更换指定版本板卡。
  3. 排查本站同批次装置。
```

- 每条 ≤ 60 字
- 编号必须 1. 2. 3.
- 三条以内；超过三条视为冗余

### 推理追溯字段（活页机制）

```
● 推理追溯：[点击展开] / data:confidence 0.83 / rules:B1,A2
```

点击展开后显示：

```json
{
  "matched_rules": ["B1-通道一无有效帧持续>24h", "A2-装置报警严重告警"],
  "fahp_score": 78.5,
  "raw_rule_level": "危急",
  "final_level": "危急",
  "confidence": 0.83,
  "data_sources": [
    "保护装置告警记录.json#alarms[1-9]",
    "保护装置实时运行状态.json#line_protections[0]",
    "皋城变...压板.json#devices[0]"
  ],
  "weight_basis": "w_A=0.40 w_B=0.27 w_C=0.20 w_D=0.13"
}
```

## 四、典型格式（完整示例）

```
【危急预警】风险简报
● 风险等级：🔴 危急
● 影响范围：500kV 皋城变 → 皋文 5325 线 → 第一套 PCS-931SA-G
● 风险概述：装置 CPU 板卡严重故障告警持续 6 小时未复归，
           同期通道一无有效帧告警 30 条/24h，存在主保护拒动风险。
● 核心建议：
  1. 立即通知调度，按调度指令退出该保护或投运备用保护。
  2. 启动紧急消缺流程，参照厂家技术说明书更换指定版本板卡。
  3. 排查本站同批次 PCS-931 装置，加强同型号装置巡视。
● 推理追溯：[展开]  confidence: 0.83  rules: A1, B1
```

## 五、批量简报（多装置时使用）

当评估对象为全网/全厂时，**先按"危急 → 严重 → 一般 → 提示"分组**：

```
═══════════════════════════════════════════════
  皋城变 / 2026-07-11 17:48 风险简报（共 8 套异常装置）
═══════════════════════════════════════════════

【危急预警】（1 套）
[简短简报 1]

【严重预警】（3 套）
[简短简报 2]
[简短简报 3]
[简短简报 4]

【一般预警】（4 套）
...
```

每条简短简报保留三行（风险概述 + 一条核心建议 + 推理追溯指针）。

## 六、不允许出现的写法

- ❌ "请尽快处理"  ← 不可执行
- ❌ "建议加强巡视"  ← 不可量化
- ❌ "可能存在风险"  ← 不可追溯
- ❌ "等级：紧急"  ← 不在四级之内
- ❌ "1." 但空缺正文  ← 占位符必须删除

## 七、HTML 活页生成规则

简报 HTML 文件同时生成：

```
out/<时间戳>/briefing.html
├── 每条规则命中 = 一个可展开 <details>
├── 数据快照 = 一个 JSON 折叠
└── 整体 confidence + weight = 表格在结尾
```

`scripts/run_risk_assessment.py:render_html_briefing()` 实现。
