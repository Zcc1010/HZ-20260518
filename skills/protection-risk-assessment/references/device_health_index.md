# 装置健康度（HealthIndex）计算

> 智能诊断层的核心输出。HealthIndex ∈ [0, 100]，
> 表示装置当下"健康程度"——**越高越健康**（与风险分相反）。

## 一、指标构成

| 维度 | 权重 | 来源 |
|---|---|---|
| H1 通讯与参数采集 | 0.20 | 实时状态.oss_status / check_status |
| H2 告警密度与等级 | 0.30 | 历史告警（最近 7d） |
| H3 压板一致性与告警一致性 | 0.15 | 压板 JSON + 实时状态 |
| H4 定值稳定性 | 0.15 | 实时定值 current vs last |
| H5 检修与服役年限 | 0.20 | 台账 |

**HealthIndex = 100 - sum(权重 × 子项扣分)**

## 二、子项扣分明细

### H1 通讯与参数采集

| 子项 | 扣分 |
|---|---|
| `check_status=参数异常` | -30 |
| `oss_status ≠ 运行中` | -20 |
| 模拟量 `运行=0` | -10 |

### H2 告警密度与等级（最近 7 天）

| 子项 | 扣分 |
|---|---|
| 每条严重告警（未复归） | -8 |
| 每条严重告警（已复归） | -3 |
| 每条运行异常（未复归） | -4 |
| 每条运行异常（已复归） | -1 |
| 任意告警密度 > 50 条/7d | -25（额外） |

### H3 压板一致性

| 子项 | 扣分 |
|---|---|
| 软压板与硬压板信号不一致（同名项 value 矛盾） | -20 |
| 数字量"保护检修状态=1"且保护仍"投入" | -25 |
| 远方修改定值软压板=1 且未关联变更记录 | -10 |

### H4 定值稳定性

| 子项 | 扣分 |
|---|---|
| 关键定值（差动动作电流、阻抗定值、电抗定值） `current_value ≠ last_value` | -15 |
| 关键定值 `current_value` 触界 ≥ 95% 量程 | -8 |
| 关键定值 `last_value=0` 异常 | -8 |

### H5 服役年限

| 子项 | 扣分 |
|---|---|
| 距上次检修 > 6 年（220kV 及以上） | -25 |
| 距上次检修 > 4 年（110kV 及以下） | -25 |
| 服役 > 12 年 | -10 |
| 投运日期缺失 | -5（数据完整度扣分） |

## 三、健康度与风险等级的耦合

- HealthIndex < 40 → 风险融合层强制把等级下限设为"严重"。
- HealthIndex < 20 → 风险融合层强制把等级下限设为"危急"。
- HealthIndex ≥ 80 且无规则命中 → 一般或提示。

## 四、为什么不用 ML 而用规则扣分

继电保护运维要求**可解释**——任何扣分都必须能人工重现。
神经网络"学习"出的特征对现场不可解释；规则扣分 + FAHP 满足"白盒"要求。
后续可以叠加 ML 异常检测作为辅助信号，但不出现在主结论链中。
