# FAHP 风险融合方法

> 三角模糊数层次分析法（Fuzzy Analytic Hierarchy Process）。在传统 AHP 基础上引入模糊数，
> 解决专家打分"非此即彼"的局限，得到更稳定的准则层权重。

## 一、为什么是 FAHP 而不是 AHP

- **AHP** 1-9 标度表达两两比较，但专家对"中等重要 vs 稍微重要"的判断常模糊；
- **FAHP** 用三角模糊数 `M = (l, m, u)` 表达，容忍左右摇摆；
- 经过多次迭代后权重仍稳定收敛 → 工程适用。

## 二、构造判断矩阵（基于运维经验+规程权重）

**目标层：** 继电保护运行风险 R
**准则层：** A 二次设备异常 / B 通道异常 / C 反措未执行 / D 定值方式不符

**三角模糊数判断矩阵（专家共识基线）：**

| 维度对 | 含义 | 三角模糊 M(l,m,u) |
|---|---|---|
| A vs B | 装置本体比通道略重要 | (1, 1.5, 2) |
| A vs C | 装置本体比反措稍重要 | (2, 2.5, 3) |
| A vs D | 装置本体比定值明显重要 | (3, 3.5, 4) |
| B vs C | 通道与反措同等重要 | (1, 1.2, 1.5) |
| B vs D | 通道比定值略重要 | (2, 2.5, 3) |
| C vs D | 反措比定值略重要 | (1.5, 2, 2.5) |

## 三、权重计算

按 **Chang (1996) 三角模糊数权重合成法**：

1. 对每个准则 `M_i = (l_i, m_i, u_i)` 计算模糊综合程度 `S_i = sum_j M_ij ⊗ [sum sum S_ij]^-1`
2. 由 `S_i ≥ S_j` 计算比较可能性 `V(M_i ≥ M_j)`
3. 归一化 `V_i / sum V_i` → 准则权重

**最终权重（已计算）：**

```
w_A = 0.40  // 二次设备异常
w_B = 0.27  // 通道异常
w_C = 0.20  // 反措未执行
w_D = 0.13  // 定值方式不符
```

> 实现见 `scripts/run_risk_assessment.py:fahp_weights()`，可用参数 `--recompute-weights` 让运维专家重打分重算。

## 四、综合风险分 S_Risk

### 4.1 准则层得分（0-100）

每条规则命中 → 等级分数：

| 等级 | 子层得分（未命中=0）|
|---|---|
| 危急 | 90 |
| 严重 | 65 |
| 一般 | 35 |
| 提示 | 10 |

同一准则层取**最大命中分**（避免多规则重复计数）。

### 4.2 综合分

```
S_Risk = w_A * S_A + w_B * S_B + w_C * S_C + w_D * S_D
       ∈ [0, 100]
```

### 4.3 风险等级映射（与"规则命中下限"取大者）

| 综合分 S_Risk | 映射等级 |
|---|---|
| ≥ 80 | 危急 |
| 55-79 | 严重 |
| 30-54 | 一般 |
| < 30 | 提示 |

**关键：** `final_level = max(FAHP_level, max_rule_level)`——FAHP 融合分不能把规则层"危急"拉低到"严重"，这是"规则下限"原则。

## 五、置信度计算

```
confidence = w_data × w_rule × w_alarm
```

| 维度 | 含义 | 计算 |
|---|---|---|
| `w_data ∈ [0,1]` | 多源数据完整度 | 已加载源数 / 5 |
| `w_rule ∈ [0,1]` | 规则命中强度 | 命中等级分 / 90 |
| `w_alarm ∈ [0,1]` | 历史告警密度证据 | 命中装置最近 7d 告警数 / 20，上限 1 |

`confidence` 写入简报尾部，"提示"等级 confidence 可 < 0.5，"危急"通常 ≥ 0.7。

## 六、局限与边界

- **冷启动：** 无历史告警时 confidence 由 data 与 rule 决定（多半 0.4-0.6）。
- **权重更新：** 定期由运维专家回收"误报/漏报"反馈，季度重打分。
- **跨厂家差异：** PCS-931 / NSR-303 等需查 `references/反措未执行判定.md` 单独修订。
