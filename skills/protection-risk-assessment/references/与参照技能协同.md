# 与参照技能协同

> 风险评估智能体不是孤岛——它依赖 setting-check 的说明书知识库、latent-faults 的隐患规则。

## 一、依赖关系图

```
                       +---------------------------+
                       | protection-risk-assessment |
                       +-------------+-------------+
                                     |
              +----------------------+----------------------+
              |                      |                      |
              v                      v                      v
   +-----------------------+ +------------------+ +-----------------------+
   |   setting-check       | |  latent-faults   | | safety-ticket-audit   |
   |  (说明书 / 反措库)    | |  (录波隐患规则)  | |  (倒闸操作票)         |
   +-----------------------+ +------------------+ +-----------------------+
```

## 二、与 setting-check 的协同

**复用方式：**

```python
# 加载说明书知识库
manuals_dir = "../../setting-check/references/knowledge-base/manuals"
manuals = load_manuals_by_model(device_model)  # 按厂家 / 型号
```

**典型场景：**
1. **反措未执行判定** —— 装置型号 → 加载该型号厂家文件夹下 反措文件 → 比对执行记录
2. **定值合理性** —— 引用说明书"参数范围 + 默认值"做定值漂移合理性核查
3. **跳闸矩阵** —— 变压器保护时加载 `references/transformer-trip-matrix/` 判断出口方式
4. **家族性缺陷** —— 按厂家型号 → 反事故措施文件

## 三、与 latent-faults 的协同

**调用时机：**
- 风险等级"危急"且涉及录波事件 → 主动提示"建议结合 latent-faults 复核录波"
- 历史告警含"启动"、"动作"、"跳闸" → 把装置 ID 传给 latent-faults 做深度复核

## 四、与 safety-ticket-audit 的协同

**调用时机：**
- 风险等级"危急"且建议涉及"停止遥控"、"定值修改" → 提示"按 safety-ticket-audit 流程出操作票"

## 五、本技能不重复实现的部分

为了避免与现有技能冲突，**下列能力被有意排除：**

| 能力 | 由谁负责 |
|---|---|
| 录波波形解析 | comtrade-parser |
| 录波隐患规则 | latent-faults-analysis |
| 定值单/计算书校核 | setting-check |
| 操作票审查 | safety-ticket-audit |
| 误动跳闸溯因 | over-trip-analysis |

风险评估智能体只回答"现在风险有多大 / 哪里有问题 / 怎么办"。

## 六、未来扩展点
