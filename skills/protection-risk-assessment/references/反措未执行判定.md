# 反措未执行判定规则

> 国家电网/省公司公开反事故措施、家族性缺陷治理的执行性追踪。
> 反措条目以字典形式维护，规则层按型号 + 关键词命中。

## 一、反事故措施条目格式

```yaml
- countermeasure_id: CM-2024-PCS931-BOARD-V2
  title: PCS-931 板卡升级至 V2.3
  affected_models:
    - PCS-931
    - PCS-931SA-G
    - PCS-931A-G
  issue_date: 2024-03-15
  deadline: 2024-12-31
  severity: 危急
  description: 旧批次 CPU 板卡存在寄生电容引起误动
  execution_evidence: 检修记录 / 板卡更换照片
```

## 二、内置示例条目

| 编号 | 厂家 | 型号前缀 | 缺陷 | 截止 | 严重度 |
|---|---|---|---|---|---|
| CM-2024-PCS931 | 南瑞继保 | PCS-931 | CPU 板卡寄生电容 | 2024-12-31 | 危急 |
| CM-2023-NSR303 | 国电南瑞 | NSR-303 | 光纤模块接收灵敏度劣化 | 2023-12-31 | 严重 |
| CM-2024-CSC103 | 北京四方 | CSC-103 | 零序电流采样漂移 | 2024-06-30 | 严重 |

> 上表仅为示例骨架，实际运营时由运维/反措主管部门定期追加。
> 数据格式维护在 `references/反措库.csv` 或同目录 YAML 文件中。

## 三、判定规则 C1 / C2

### C1 反措未按时执行

```
IF (device.model IN entry.affected_models)
   AND (entry.deadline < today)
   AND ("已执行" NOT IN device.maintenance_records):
   → 触发 C1，初始等级 = entry.severity
```

### C2 国网/省公司"十八项反措"

与 C1 类似但清单是另一份公开反措条目（详见知识库 `references/反措库_国网十八项.yaml`）。

## 四、扩展方式

| 来源 | 维护方式 |
|---|---|
| 国家电网公开反事故措施 | 由"反措主管部门"每季度追加 YAML 条目 |
| 省公司发文 | 单独存放 `references/反措库_省公司.yaml` |
| 厂家技术改造通知 | 厂家型号下另设子目录 |

每条新增反措必须包含：`countermeasure_id / title / affected_models / deadline / severity / description`。
