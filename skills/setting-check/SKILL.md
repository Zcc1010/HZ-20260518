# 继电保护定值校核

由 Claude/pi 直接完成定值校核，按本流程逐条分析，输出面向审查的正式校核报告。
本 SKILL.md 主讲流程；具体规则、判定标准、报告格式、约束等见 `references/`。

## 工作流

```
0、验证环境       ← 必须先做：确认工作区目录存在
     ↓
一、加载参考材料   ← 整定原则 + 校核报告模板 + 说明书（已自动链接到工作区）
     ↓
二、信息提取      ← 设备类型/电压/装置型号（从定值单）
     ↓
三、逐条校核      ← 读工作区的模板 + 整定原则
     ↓
四、生成报告      ← 校核报告.md
```

### 0. 验证环境（必做第一步）

执行任何操作前，先确认工作区目录存在：

```bash
# 确认工作区目录存在
ls "workspace/<工作区名>/"
```

工作区目录结构已自动创建，包含：
- `定值单/` — 定值单文件
- `计算书/` — 计算书文件
- `报告/` — 生成的报告
- `整定原则/` — 整定原则和上下级定值模板（自动筛选）
- `校核报告模板/` — 校核报告模板（自动筛选）
- `台账/` — 台账文件（自动筛选）
- `说明书（{厂家}-{型号}）/` — 装置说明书（自动匹配）

### 一、加载参考材料

所有参考材料已自动链接到工作区，无需手动复制：

| 目录 | 内容 | 说明 |
| --- | --- | --- |
| `整定原则/` | 整定原则 + 上下级定值模板 | 按设备类型和电压等级自动筛选 |
| `校核报告模板/` | 通用模板 + 设备专属模板 | 按设备类型自动筛选 |
| `说明书（{厂家}-{型号}）/` | 装置说明书 | 自动匹配厂家和型号 |

**工作区目录结构**：
```
workspace/<工作区名>/
├── 定值单/
├── 计算书/
├── 报告/
├── 整定原则/          ← 自动链接，按设备类型筛选
├── 校核报告模板/      ← 自动链接，通用 + 设备专属
├── 台账/              ← 自动链接，按设备类型筛选
└── 说明书（{厂家}-{型号}）/  ← 自动链接
```

**说明**：
- 整定原则和校核报告模板在进入工作区时自动链接，无需手动复制
- 说明书自动匹配厂家和型号，如果没有找到会显示提示
- 所有文件都已按设备类型和电压等级筛选，只显示相关文件

### 二、信息提取

从定值单和计算书提取：

- **设备名称**（必填）— 从定值单中的设备名称字段提取（如"电灌站线325线路"），用于报告标题 `# {厂站} {设备名称} 定值校核报告` 和基本信息表
- 设备类型（变压器/线路/母线等）— 从定值单中的设备名称和保护类型判断
- 电压等级 — 从定值单中的设备名称或PT一次值判断
- 装置型号 — 从定值单中的保护装置型号字段提取

**先读定值单（基准），再读计算书（辅助）。** 文本格式直接用 `read`，二进制格式用 `npx tsx scripts/read-file.ts "<路径>"`。

### 三、逐条校核

校核的维度、判定标准、报告结构全部以模板为准，本节不重复。操作：

1. 读 `校核报告模板/校核报告模板-通用.md`（校核维度、判定标准、报告结构）
2. 读 `校核报告模板/校核报告模板-{设备类型}.md`（设备参数、保护功能清单、上下级配合要求等设备特殊内容）
3. 读 `整定原则/{设备类型}整定原则.md`
4. 填入上下级定值时，按 `references/上下级定值单查找规则.md` 定位对侧定值单，结果写入 `workspace/<工作区名>/{设备类型}上下级定值.md`
5. 按模板逐条校核
6. 关键约束见 `references/校核关键约束.md`（依据来源、维度判定、报告写作、条款引用格式等）

### 四、生成报告

按校核报告模板（通用 + 设备专属）的结构输出到 `workspace/<工作区名>/校核报告.md`。

报告中的"问题"和"提醒"加粗显示，只列不通过的项。

## ⚠️ 文件名安全规则

1. **所有 bash 命令中的文件路径必须用引号包裹**
2. **绝对不要在文件名中间加空格**：`北湖变131定值单.xls` 正确，`北湖变 131 定值单.xls` 错误
3. **英数字与中文之间不加空格**：`220kV变压器` 正确，`220kV 变压器` 错误

## 关键约束（速查）

完整版见 `references/校核关键约束.md`。速查：

- **依据来源**：以定值单为基准、计算书为辅；上下级定值必须来源对侧定值单
- **维度判定**：退出的功能维度三/四/五判"不适用"，维度一/二仍校核
- **维度二逐条**：整定原则对该功能的每条条件都要交代（强弱分级 + 校核结果或数据缺失说明），不能只写"取值一致"
- **维度一 vs 原则**：定值单与计算书不一致但满足整定原则 → **提醒**（不是问题）；违反强约束才判"问题"
- **维度五 vs 时间**：对侧定值单**日期较早不作为问题**判据，除非明确标注"作废"；缺少对侧定值单但对侧性质不明（可能是纯负荷站）→ **提醒**，不是问题
- **报告写作**：只写问题/提醒；不写"符合xxx要求"；不出现技术过程术语
- **条款引用**：`章节号.小节号.(子项号)`，如 `三.2.(3)`、`二.11`
