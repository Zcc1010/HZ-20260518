# 上下级定值单查找规则

> 校核时定位对侧 / 上级 / 下级设备的定值单路径。配合 `整定原则/{设备类型}上下级定值（模板）.md` 使用。
> 不得从计算书引用上下级定值（AGENTS.md 强约束）。

## 一、总则

### 1. 先找后读，避免上下文溢出

定值单库全量数百 MB，过早批量读取会撑爆上下文窗口。**严格遵守"先定位、再单文件读取"**：

1. **只查索引**：用 `ls` / `find` / `grep -l` 定位目标文件路径，不读取内容
2. **找到目标再读**：用 `npx tsx tools/read-file.ts "<具体路径>"` 读单个定值单
3. **禁止批量预读**：禁止 `cat` / `read` 整个厂站文件夹的所有 `.xls`，禁止把台账 CSV 整表读入后再多次扫描
4. **台账只读一次**：找到目标行后记下「编号 / 名称 / 首厂站 / 末厂站 / 电压等级」即丢弃，不必保留原始内容

工具选择：

| 操作 | 工具 |
| --- | --- |
| 目录浏览 / 列表 | `ls` |
| 文件名关键字搜索 | `grep -l "<关键字>" <路径>` |
| 单文件读取（任意格式） | `npx tsx tools/read-file.ts "<路径>"` |

### 2. 特殊运行方式以用户说明为准

旁路代供、临时转供电、检修方式等特殊运行方式下，拓扑与默认规则不一致。**用户在指令中明确说明的上下级关系，覆盖本文档所有默认判定**（规则 2 的 Step B / Step D、规则 4 的 Step 4 均以用户说明为准）。

判断顺序：

1. 用户已明确上下级 → 直接采用用户说明，跳过规则 2 / 4 的台账与电压判定
2. 用户未说明 → 按本文档规则自动判定
3. 校核报告中标注「按用户说明 / {说明内容}」以体现依据

## 二、数据来源

| 来源 | 用途 |
| --- | --- |
| 定值单文件名称 | 设备类型识别、电压等级匹配 |
| 定值单库文件夹层级 | 厂站归属与电压等级（目录名带电压前缀，如 `220kV宝桥变/`） |
| `resources/account/线路台账.xls` | 仅线路用：编号 / 名称 / 首厂站 / 末厂站 / 电压等级 |

```
setting-orders/滁州公司/
├── 220kV{站名}/         ← 直属站
│   ├── *.xls              ← 本站定值
│   └── {用户变名}/        ← 混入的用户变定值（如 唐龙风场/、吉电光伏/）
├── 110kV{站名}/
├── 500kV{站名}/
├── 35kV{站名}/          ← 直属 35kV 站
└── {天长|来安|全椒|定远|城郊|明光}公司/
    └── {站名}/          ← 子公司下属
```

层级识别约定：
- 直属站 / 子公司站：定值单以 `.xls` 直接放在站文件夹下
- 用户变：定值单放在站文件夹下的子文件夹里（子文件夹名 = 用户变名，如风场 / 光伏 / 用户厂）

## 三、规则

### 规则 1：同厂站内设备（最快）

适用：上级 / 下级主变、母联分段另一段母线、电容器 / 电抗器 / 站用变 / 接地变的母线、电抗器上级电源线路。

路径：当前厂站文件夹 → 找含目标关键词的定值单 → 按电压等级筛选。

判断要点：
- 厂站有多个电压等级时（如 220/110/35/10kV 共站），按电压等级匹配：找某侧保护就取对应电压的定值单
- 多台主变按定值单名区分（`#1主变` / `#2主变`）
- 同一保护常有多套（如"第一套""第二套"），按校核对象套数取
- 旁路（`旁路{编号}开关.xls`）可临时替代同电压线路 bay
- **跳过用户变子文件夹**：站文件夹下的子目录（`{用户变名}/`）是用户变定值，与本站无关，不要混入检索结果

### 规则 2：线路（最复杂）

适用：任何电压等级线路的"上级"或"下级"对侧定值单。

**Step A**：从定值单提取线路名 / 编号 → 在 `resources/account/线路台账.xls` 按编号或名称查 → 拿到「首厂站」「末厂站」。

**Step A.1：用户终端线路识别**。若「首厂站」==「末厂站」（同名），这是用户终端专线，无下级设备，下级定值单维度不适用（不适用记录在上下级定值.md 中注明）。上级仍是本厂站母线 / 主变，按规则 1 处理。

**Step B：T 接识别与展开**。T 接线路有一端是 T 接点（厂站名以 `T` 开头：`T鲁`、`T10`、`T#4`、`T36` 等），不是真实厂站。处理：

1. 检查线路两端，任一端以 `T` 开头 → 这条线是 T 接线路
2. 在线路台账中以该 T 节点为关键字检索（限同电压等级），找到所有以它为端点的其他线路
3. 这些线路的另一端就是 T 节点实际连接的真实厂站
4. 递归展开，直到所有端点都是不带 `T` 的真实厂站名

示例：`鲁梓610` 在台账中是 `鲁肃变 ↔ T10`；以 `T10` 为关键字再查台账，命中 `T韭山（T10 ↔ 韭山变）` 和 `T梓树（T10 ↔ 梓树变）`；展开为三站 T 接：`鲁肃变`、`韭山变`、`梓树变`。

**Step C**：到定值单库逐个搜索真实厂站，确定各自实际电压等级——厂站可能跨电压等级（如 220kV 宝桥变带 110kV 出线 bay）。

**Step D**：按电压等级判定上下级：

| 本侧厂站电压 | 对侧厂站电压 | 对侧角色 |
| --- | --- | --- |
| 110kV | 220kV | 上级（220kV 侧定值 = 上级定值） |
| 110kV | 110kV | 非上级，需另行追溯供电电源 |
| 35kV | 110kV | 上级（110kV 侧定值） |
| 10kV | 110kV | 上级（110kV 侧定值） |
| 220kV | 500kV | 上级（500kV 侧定值） |

**Step E**：定位文件。在定值单库下，对侧厂站文件夹内 grep 编号或线路名片段；命中即取 `开关.xls` 或 `线路.xls`。

匹配技巧：
- 同一对厂站间常有多条平行线（如 `栗河622` 和 `栗古621` 是 `古河变 ↔ 黄栗树变` 的两条独立 110kV 线路，长度也不同），grep 时必须用编号精确定位，不能只按厂站名

### 规则 3：变压器

定值单文件分组（同一主变）：
- 电量：`#N主变第一套` / `#N主变第二套`（按套数取）
- 高/中/低后备：`#N主变{高|中|低}后备（第N套）`
- 非电量：`#N主变非电量`
- 差动：`#N主变差动`

侧别按电压等级：厂站最低电压等级 = 低压侧。

### 规则 4：已知厂站反向找上级

适用：校核某厂站内设备（主变高压侧、母线、bay 等）需要找上级电源厂站时——起点是厂站而非线路。

处理：

1. 识别该厂站的最高电压等级 V_max（看定值单库目录前缀 / 该厂站最高电压定值）
2. 在 `resources/account/线路台账.xls` 中筛选「电压等级 = V_max」+「首厂站或末厂站 = 已知厂站名」的所有线路
3. 命中线路的另一端 = 上级厂站候选
4. 移交规则 2 的 T 接展开（Step B）与电压判定（Step D），确定真实上级

例：北湖变是 110kV 站 → 在台账筛 110kV + 北湖变 → 命中 `建北554,建阳变,北湖变,110` → 另一端建阳变（220kV）= 上级。

## 四、输出

将找到的对侧 / 上下级定值单信息填入工作区 `{设备类型}上下级定值.md`：

```
| 保护功能 | 一次值 | 时间(s) | 定值单来源（相对路径）|
```

若定值单正文中有地调 / 省调下达的编号，一并记录在「备注」列（编号格式因地调 / 省调而异，不预设）。

## 五、注意事项

- **所有文件路径必须用双引号**（AGENTS.md）
- **禁止猜测文件名**，先 `ls` 看复制粘贴
- **英数字与中文之间不加空格**：`220kV变压器` 不能写成 `220kV 变压器`
- **不得从计算书引用上下级定值**；仅有计算书引用而无定值单 → 维度五判"问题"
- **资料不足时具体说明**缺什么、在哪份资料缺失、因此不能核定什么（不要编造）
