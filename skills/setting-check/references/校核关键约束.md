# 校核关键约束

> 校核过程中必须遵守的硬性约束。这些是行业与工程规范层面的要求，不允许因"看起来 OK"而跳过。

## 1. 依据来源约束

- 校核必须严格依据加载的整定原则原文，不得凭印象判断
- 以定值单为基准值与整定计算原则比对；计算书作辅助参考
- 上下级定值信息必须来源于对侧设备的定值单，不能从计算书引用。仅有计算书引用而无定值单 → 维度五判"问题"
- 当计算书缺少明确整定结论但定值单存在明确整定值时，可将定值单作为取值来源，结合计算书公式和原则完成校核

## 2. 维度判定约束

- 退出的保护功能（控制字=0）：维度三/四/五判"不适用"；维度一/二仍需校核
- 未投入的功能不校核其取值，定值单列只写"退"
- 投入的功能在定值单列写关键定值和控制字信息
- 定值单与计算书不一致时归入提醒事项（除非违反强约束）

## 3. 校核维度与判断依据

校核概览中每个保护功能检查 5 个维度，各维度结论为：**一致** / **问题** / **提醒** / **不适用**。

### 维度一：计算书（定值单与计算书一致性）

| 结论      | 判断依据                                                                                                                                                 |
| ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **一致**  | 定值单各项定值（电流/电压/时间）和控制字状态与计算书整定结论吻合                                                                                                                    |
| **问题**  | ① 定值单的值**违反强约束**（计算书已校核过原则要求，定值单超出原则允许范围）<br>② 控制字逻辑值不一致（计算书投入但定值单退出，或反之）<br>③ 定值单中的设备信息、参数与计算书不一致（CT变比、容量等关键参数）<br>④ 计算书中有明确的计算或取值错误（公式错误、系数取错等） |
| **提醒**  | ① 定值单整定值与计算书整定值不一致，但仍在整定原则允许范围内（**不构成"问题"**——计算书只是参考，整定原则才是硬约束；定值单与计算书不同但原则允许，是正常工程做法）<br>② 计算书未涉及该功能但定值单有取值（视为补充整定）                                                                                          |
| **不适用** | ① 计算书没有对该功能进行计算                                                                                                                                      |

> **关键判别**：定值单 vs 计算书不一致 → **先看是否满足整定原则**。满足 → 提醒（不是问题）；不满足（违反强约束）→ 问题。不要把"定值单≠计算书"直接等同于"问题"。

### 维度二：整定原则（是否符合整定原则条款）

**校核方法（逐条枚举，不可只看定值单=计算书就写"一致"）**：

整定原则对该保护功能通常列出 3-5 条整定条件（如"躲变压器低压侧故障"、"小方式灵敏度 ≥ X"、"与上级 XX 段配合"等）。校核时**必须**逐条处理：

1. **枚举条件**：从整定原则原文中摘出该保护功能的所有整定条件
2. **标记强弱**：根据用词判定约束强度
   - 强约束：含"应""不得""必须""严禁""≥""≤"等
   - 弱约束：含"宜""可""建议""一般""推荐"等
3. **判断数据可用性**：校核该条件需要的输入（短路电流、对侧定值、配合系数等）是否在定值单/计算书/说明书/上下级定值单中能取到
4. **逐条计算或说明缺失**：能算的代入公式算出限值，对比定值单；不能算的明确说明缺什么
5. **聚合判定**：根据各条结论按下列规则聚合

| 结论 | 判断依据 |
|------|---------|
| **一致** | 所有适用的强约束条款**已校核通过**；弱约束条款全部满足或未违反 |
| **问题** | ① 强约束条款**已校核但违反**：灵敏系数低于最低要求、配合系数低于最低值、时间级差低于最低值<br>② 功能投退状态违反原则强约束要求<br>③ 定值超出原则规定的允许范围<br>④ 强约束条件**数据缺失且无法补全**：计算书缺少关键短路电流、对侧定值单缺失等导致强约束无法校核（视为违反强约束） |
| **提醒** | ① 强约束全部满足，但弱约束未满足<br>② 弱约束条件数据缺失（不影响保护正确性，可后续完善）<br>③ 原则推荐某取值，实际取值偏离但在允许范围内 |
| **不适用** | ① 该保护功能不适用于此设备类型或电压等级<br>② 整定原则中没有提及该功能 |

### 维度三：保护原理（控制字配置是否符合说明书保护逻辑）

| 结论 | 判断依据 |
|------|---------|
| **一致** | 控制字配置符合说明书描述的保护逻辑，各控制字之间无矛盾 |
| **问题** | ① 控制字配置导致保护逻辑与说明书不符：如经方向但方向控制字退出、经复压但复压控制字退出<br>② 控制字配置自相矛盾：如"过流Ⅰ段经方向"=1但"过流Ⅰ段经电压"=0导致方向元件无电压量<br>③ 方向指向与系统接线方式不符：如双电源线路方向指向错误 |
| **提醒** | ① 控制字配置逻辑上可行但非最优：如可投复压闭锁提升选择性但实际未投<br>② PT断线后的行为（退出方向/退出保护）与现场需求可能不完全匹配 |
| **不适用** | ① 该功能未投入（退出时控制字配置不参与保护逻辑，无需校核）<br>② 说明书中无该功能的控制字说明（说明书缺失或型号不匹配） |

### 维度四：定值项（定值单项目是否与说明书定值清单一一对应）

| 结论 | 判断依据 |
|------|---------|
| **一致** | 定值单项目与说明书定值清单一一对应，无多余无遗漏，定值在说明书规定范围内 |
| **问题** | ① 定值单缺少说明书要求的必设定值项（如缺少零序CT二次值）<br>② 定值单中有说明书不存在的定值项（装置型号/版本可能匹配）<br>③ 定值超出说明书规定的定值范围（如说明书规定0.05~20Ie，定值单取0.01Ie）<br>④ 定值步长不符合说明书规定 |
| **提醒** | ① 定值单项目与说明书清单基本对应，个别项名称表述略有差异但不影响识别<br>② 装置版本与说明书版本不完全一致，个别定值项可能有差异 |
| **不适用** | ① 说明书缺失，无法核对<br>② 该功能退出 |

### 维度五：上下级定值（上下级信息收集完整性）

| 结论 | 判断依据 |
|------|---------|
| **一致** | 需要配合的上下级定值信息已收集完整（且有对侧定值单来源） |
| **提醒** | ① 缺少对侧定值单，但**对侧可能是纯负荷站/用户站**（无独立电源或不需要保护配合）——需确认对侧性质后再下结论<br>② 对侧定值单已收集但**对侧投退状态不明**（如重合闸投退、是否检同期）——需现场确认<br>③ 对侧定值单来源时间较早，但**未被明确标记作废**——按"有效"处理，不以此判问题（定值单日期不作为有效性依据，除非文档明确标注"作废"） |
| **问题** | ① 对侧**确认是电源站**（有发电机/上网线路/并网），但缺少对侧定值单，无法完成保护配合校核 |
| **不适用** | ① 该功能不需要与上下级配合（如过负荷告警、低频减载）<br>② 该功能未投入 |

> `{设备类型}上下级定值.md`（如 `10~20kV线路上下级定值.md`）中应列出所有需要配合的对侧设备定值项，有定值单来源的填入实际值，没有的留空。维度五逐行核对：有对侧定值单且完整 = 一致；仅有计算书引用但**对侧性质不明** = 提醒；明确有电源的对侧缺定值单 = 问题。
> **上下级定值信息必须来源于对侧设备的定值单。** 计算书中引用的对侧保护定值不作为有效来源，不得填入。
> **不要把"定值单日期较早"作为有效性判断依据**——除非该定值单明确标注"作废"或"废止"，否则视为有效。若需提示定值单较旧，应作为"提醒"事项注明"建议确认是否为现行有效定值"，**不**因此判"问题"。
> 配合系数、时间级差等配合正确性的校核在维度二（整定原则）中进行。

## 4. 报告写作约束

- 报告中只写有问题/提醒的内容，符合要求的不写
- 不出现技术过程术语（脚本、规则ID、JSON、路由、实例、SKILL、模板等）
- 不编造计算书中不存在的对应值
- 资料不足时具体说明缺什么、在哪份资料、因此不能核定什么
- 存在问题和提醒事项中不写"取值原则""建议"等子标题，直接写问题描述
- 不写"符合xxx要求"之类的正面描述

**维度二的报告写法**（关键）：

对每个保护功能做维度二校核时，**不能**只写一句"取值与整定原则一致"就完事。**内部**必须按整定原则列出的所有条件逐条算（校核 5 步流程），但**报告里只写问题/提醒**（符合已有"只写问题/提醒"规则，满足/不适用的条件不写）。

**例**（过流 II 段 800A/0.3s，5 条条件中 2 条数据缺失）：

报告里只写：

> **过流 II 段**
> - 计算书未给出变压器低压侧三相短路电流，躲变压器低压侧故障电流（强约束）无法校核。
> - 计算书未给出小方式末端故障电流，灵敏度校验（强约束）无法校核。

不写"条件 3 与上级配合满足"（因为满足），不写"条件 4、5 终端线不适用"（因为不适用）。

**判定逻辑**：5 条里只要有任何一条强约束数据缺失或被违反，维度二整体就判"提醒"或"问题"（按强弱和数量聚合），并把不通过的那几条写到报告里。

## 5. 条款引用格式

引用原则原文条款时使用"章节号.小节号.(子项号)"格式：
- `二.11` — 第二章总则第11条（时间级差）
- `三.2.(3)` — 第三章第2节第(3)项（二次谐波制动系数）

## 6. 校核概览表格

- 退出的功能定值单列只写"退"
- 表中"问题"和"提醒"加粗显示
- 表格下方不写说明文字
- 概览/提醒/问题中"提醒"和"问题"加粗显示
