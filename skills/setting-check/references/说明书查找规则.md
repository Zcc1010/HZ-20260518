# 说明书查找规则

> 校核前需把装置说明书复制到工作区。脚本（`tools/copy-templates.ts`）默认按"型号完全匹配"找，找不到时**不要直接判"缺失"完事**——可以按"型号前缀 + 设备类型"推断厂家，去对应目录找同系列型号。

## 一、目录约定

```
resources/manuals/
├── {厂家}/                  ← 8 家：国电南自 / 国电南瑞 / 南瑞科技 / 上海思源 / 南瑞继保 / 长园深瑞 / 北京四方 / 许继电气
│   └── {设备类型}/          ← 线路保护 / 变压器保护 / 母线保护 / 电容器保护 / 电抗器保护 / 站用变保护 / 断路器保护
│       └── {型号}/          ← 完整装置型号，如 PCS-9611A、NSR-378T2
│           ├── {型号}{类型}说明书.md
│           └── images/
```

## 二、查找流程

### 步骤 1：型号完全匹配

脚本默认行为：`find resources/manuals -type d -name "<型号>"` 精确匹配。命中即复制。

### 步骤 2：型号前缀 + 设备类型推断（精确未命中时）

定值单里的"保护装置型号"是核心信息。取**型号前面的英文字母部分**（前缀），对照下表推断厂家。

| 型号前缀 | 厂家 | 主要设备类型 |
| --- | --- | --- |
| `PCS-`, `RCS-` | 南瑞继保 | 线路 / 母线 / 变压器 / 电容器 / 站用变 / 断路器 |
| `PSL-`, `PSC-`, `PST-`, `SGT-`, `SGR-`, `SGB-` | 国电南自 | 线路 / 母线 / 变压器 / 电容器 / 电抗器 |
| `NSR-` | 国电南瑞 / 南瑞科技 | 见下表细分 |
| `UDL-` | 上海思源 | 线路 / 母线 |
| `PRS-`, `ISA-`, `BP-` | 长园深瑞 | 线路 / 母线 / 变压器 / 电抗器 |
| `CSC-` | 北京四方 | 线路 / 母线 / 变压器 |
| `PAC-`, `WBH-`, `WDLK-`, `WMH-` | 许继电气 | 母线 / 变压器 |

#### `NSR-` 细分（两家都有，需按设备类型区分）

| 设备类型 | 厂家 | 关键型号特征 |
| --- | --- | --- |
| 变压器保护 | **国电南瑞**（NSR-378T2 独有） | `NSR-378T2` 一定在国电南瑞 |
| 变压器保护 | 南瑞科技 | `NSR-378T`（不带 2） |
| 母线保护 | 两家都有 | 选国电南瑞优先（NSR-322CG 是国电南瑞的） |
| 线路保护 | 两家都有 | 选国电南瑞优先（NSR-302G 是国电南瑞的） |
| 电容器 / 电抗器保护 | **南瑞科技**独有 | `NSR-3620A`、`NSR-377` |

### 步骤 3：候选目录搜索

确定厂家 + 设备类型后，在对应目录下找：

```bash
ls "resources/manuals/<推断的厂家>/<设备类型>/" | grep "<型号前缀>"
```

例如型号 `PAC-8211A`、设备类型线路保护（虽然实际是 35kV 线路）：

```bash
ls "resources/manuals/许继电气/线路保护/" 2>/dev/null
ls "resources/manuals/许继电气/变压器保护/" 2>/dev/null
```

若**同系列的其他型号**（如 `PAC-860` 系列）存在，可参考其说明书（同一厂家控制字命名约定相似）。

### 步骤 4：仍找不到时的报告

校核报告中这样写：

> 说明书缺失：保护装置型号 PAC-8211A（许继电气 / 线路保护或变压器保护）。当前说明书库中未收录该型号；建议从厂家技术支持渠道获取或参考同厂家同类型装置的说明书。

## 三、查找决策速查

```
型号？ → 取前缀
  ├─ PCS-/RCS- → 南瑞继保
  ├─ PSL-/PSC-/PST-/SGT-/SGR-/SGB- → 国电南自
  ├─ NSR- → 看设备类型
  │    ├─ 变压器 + 含 T2 → 国电南瑞
  │    ├─ 电容器 / 电抗器 → 南瑞科技
  │    └─ 其他 → 国电南瑞（优先）
  ├─ UDL- → 上海思源
  ├─ PRS-/ISA-/BP- → 长园深瑞
  ├─ CSC- → 北京四方
  └─ PAC-/WBH-/WDLK-/WMH- → 许继电气
```
