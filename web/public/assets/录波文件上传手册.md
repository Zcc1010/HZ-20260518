# 录波文件上传手册

## 一、文件目录结构要求

上传的 ZIP 压缩包内必须包含以下目录结构，否则系统无法正确识别和解析。

```
事故名称.zip
├── 保护录波/                  ← 保护装置录波（必须）
│   ├── 厂站A/
│   │   ├── 第一套保护装置型号/
│   │   │   └── 录波文件（.cfg/.dat/.hdr 或 .zwav 或嵌套 .zip）
│   │   └── 第二套保护装置型号/
│   │       └── 录波文件
│   └── 厂站B/
│       └── ...
└── 故障录波/                  ← 故障录波器录波（如有）
    ├── 厂站A/
    │   └── 录波文件（直接放置或嵌套 .zip）
    └── 厂站B/
        └── ...
```

## 三、目录层级说明

### 第一层：事故名称
- 即 ZIP 文件名，可自定义，例如「2025年5月22日 朗贺4V58线.zip」

### 第二层：录波类型（二选一或两者都有）

| 目录名 | 说明 |
|--------|------|
| `保护录波` | 各保护装置的录波数据 |
| `故障录波` | 故障录波器的录波数据 |

### 第三层：厂站名称
- 即变电站名称，例如「朗峰变」「贺庄变」
- 每个厂站一个文件夹

### 第四层（仅保护录波）：套别 + 装置型号
- 即保护装置套别及型号，例如「第一套保护PCS931A」「第二套保护WXH-803A-DA-G」

### 第五层：录波文件
- 录波数据文件，支持以下格式：
  - **标准 COMTRADE 文件**：`.cfg` + `.dat` + `.hdr`
  - **南瑞 zwav 格式**：`.zwav`（单一文件，内含完整录波数据）
  - **嵌套 ZIP 包**：`.zip`（内部包含 COMTRADE 文件）

## 四、支持的文件格式

| 格式 | 说明 | 来源 |
|------|------|------|
| `.cfg` + `.dat` + `.hdr` | 标准 COMTRADE 格式 | 通用 |
| `.zwav` | 南瑞继保录波文件 | 南瑞保护装置 |
| 嵌套 `.zip` | 内含 COMTRADE 文件的压缩包 | 各厂家导出 |



## 五、打包步骤

1. 按上述目录结构整理好文件
2. 进入事故名称文件夹，全选「保护录波」「故障录波」
3. 右键压缩为 ZIP 格式（如「2025年5月22日 朗贺4V58线.zip」）
4. 上传到录波解析系统

> **注意**：打包是在「保护录波」「故障录波」目录下全选打包。不要 zip 解压之后多一层装置名称的目录，否则会解析失败。

## 六、常见问题

### Q: 上传后提示「未找到任何保护装置文件」
A: 请检查 ZIP 内是否包含 `保护录波` 或 `故障录波` 目录，且目录名不能有误（如写成「保护录波数据」将无法识别）。

### Q: 中文目录名解压后乱码
A: 系统已内置 GBK 编码修复，如仍出现乱码，请确认 ZIP 是在 Windows 系统下打包的。

### Q: .zwav 文件能否直接解析？
A: 可以。系统支持南瑞继保的 .zwav 格式，会自动提取其中的录波数据。

### Q: 嵌套 ZIP 包能否解析？
A: 可以。系统会自动解压嵌套的 ZIP 文件并解析其中的 COMTRADE 数据。

### Q: 能否只上传故障录波或只上传保护录波？
A: 可以，两者有其一即可。系统会分别扫描 `保护录波` 和 `故障录波` 目录。
